# التحضير للمستقبل

هذا الفولدر مخصص لتحويل حزمة قوالب المونديال إلى أساس قابل لإعادة الاستخدام مع مسابقات أخرى مثل دوري أبطال أوروبا، الدوري الإسباني، كأس آسيا، أو أي بطولة لاحقة.

الفكرة ليست نسخ القوالب وتغيير الاسم فقط. المطلوب أن تصبح الحزمة قابلة للتغذية من أي جسر بيانات بنفس المنهج:

- مسابقة.
- موسم.
- فرق وأعلام وشعارات.
- مباريات مباشرة وقادمة ومنتهية.
- تفاصيل مباراة.
- تشكيلات.
- إحصائيات.
- هدافون ولاعبون.
- أدوار إقصائية أو جدول دوري حسب نوع المسابقة.

## محتويات الفولدر

- `01-خريطة-القوالب-الحالية.md`
  - تنظيم قوالب المونديال قالباً قالباً وما يحتاجه كل قالب.
- `02-عقد-الجسر-لمسابقة-جديدة.md`
  - شكل البيانات المطلوب من أي جسر جديد حتى تعمل القوالب بدون إعادة بناء كبيرة.
- `03-خطة-تحويل-الحزمة-الى-مسابقة-اخرى.md`
  - خطوات عملية لإضافة مسابقة جديدة.
- `04-دفعات-التوسع-المقترحة.md`
  - دفعات التطوير القادمة لتكبير المكتبة وتقليل الأخطاء المتراكمة.

## القاعدة الأساسية

أي مسابقة جديدة يجب أن تدخل عبر محول بيانات موحد، ثم تستخدم نفس حقول القوالب:

- `homeTeam`, `awayTeam`, `homeCode`, `awayCode`
- `homeScore`, `awayScore`, `matchStatus`, `minute`
- `playersJson`, `eventsJson`, `statsJson`
- `statPossessionHome/Away`, `statPressureHome/Away`, `statPassHome/Away`
- `groupsJson`, `scorersJson`, `bracketJson`

بهذا الشكل نستفيد من نفس تصميمات المونديال ونغير فقط هوية المسابقة ومصدر البيانات.
