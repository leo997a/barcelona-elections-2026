# خطة تحويل الحزمة إلى مسابقة أخرى

هذه الخطة تستخدم عند إضافة بطولة جديدة مثل دوري أبطال أوروبا أو الدوري الإسباني.

## المرحلة 1 - تعريف المسابقة

1. إنشاء `CompetitionProfile`.
2. تحديد نوع المسابقة:
   - `league`: جدول دوري فقط.
   - `cup`: أدوار إقصائية فقط.
   - `group_knockout`: مجموعات ثم أدوار إقصائية.
3. تحديد ألوان الهوية.
4. تحديد الشعار والاختصارات.

## المرحلة 2 - الجسر

1. إضافة action أو endpoint يعيد `CompetitionSnapshot`.
2. إضافة action أو endpoint يعيد `CompetitionMatchDetails`.
3. إضافة fallback يدوي أو cache عند فشل المصدر المباشر.
4. عدم تمرير اسم المصدر إلى القوالب.

شكل مقترح:

```text
/api/competition?action=overview&competition=champions-league&season=2026
/api/competition?action=match-details&competition=champions-league&matchId=...
/api/competition?action=scorers&competition=champions-league&season=2026
```

## المرحلة 3 - محول الحقول

1. تحويل `CompetitionFixture` إلى حقول مباراة.
2. تحويل `CompetitionMatchDetails` إلى:
   - `playersJson`
   - `eventsJson`
   - `statsJson`
   - حقول `stat...Home/Away`
3. تحويل الترتيب إلى `groupsJson` أو `leagueTableJson`.
4. تحويل الأدوار الإقصائية إلى `bracketJson`.

## المرحلة 4 - الهوية البصرية

لا نعيد كتابة القوالب. نضيف preset هوية:

- اسم المسابقة.
- الألوان.
- شكل الأقواس والخلفيات.
- النصوص الافتراضية.
- شعار المسابقة.

أمثلة:

| المسابقة | الهوية |
|---|---|
| دوري أبطال أوروبا | أزرق داكن، نجوم، أقواس ضوئية |
| الدوري الإسباني | ألوان أندية، أسلوب جدول دوري |
| كأس آسيا | أحمر/ذهبي أو حسب البطولة |

## المرحلة 5 - تفعيل القوالب المناسبة

| نوع المسابقة | القوالب المهمة |
|---|---|
| دوري | scoreboard, scorebug, match stats, league table, top scorers |
| مجموعات وإقصائيات | group wall, group table, bracket, match announcement |
| كأس فقط | bracket, full-time, match report, prediction |

## المرحلة 6 - الاختبارات

كل مسابقة جديدة تحتاج اختبارات ثابتة:

- كل قالب ظاهر في المكتبة.
- كل قالب له renderer branch.
- كل قالب له تصنيف.
- كل endpoint يعيد fallback عند فشل المصدر.
- كل حقل إحصائي مهم يتحول إلى `stat...Home/Away`.

## المرحلة 7 - النشر

1. بناء المشروع.
2. دفع GitHub.
3. انتظار هوستنغر أو منصة النشر.
4. فحص بصمات القوالب الجديدة داخل ملف الواجهة الحي.
5. حفظ تقرير عربي في `reports-ar`.
