# تقرير مرحلة: تعريب نصوص غرفة التحكم وروابط القوالب

التاريخ: 2026-05-29  
اسم المرحلة: ARABIC-CONTROL-ROOM-LINK-LABELS-001  
نوع المرحلة: صيانة UX نصية صغيرة  
الملفات المعدلة:

- `pages/Operator.tsx`
- `pages/Library.tsx`
- `pages/Editor.tsx`
- `pages/Home.tsx`
- `pages/Integrations.tsx`

## 1. الهدف

بعد مرحلة روابط القوالب الخاصة، بقيت كلمات إنجليزية واضحة في الواجهة اليومية:

- `Program OBS`
- `TAKE IN`
- `TAKE OUT`
- `OFF AIR`
- `ON AIR`
- `Single Program`
- `Copied`
- `Edit Link`
- `Chroma`
- بعض أوامر Stream Deck داخل صفحة الربط الخارجي.

هذه الكلمات كانت تظهر في غرفة التحكم، المكتبة، المحرر، والصفحة الرئيسية. المرحلة الحالية عالجتها كنصوص واجهة فقط.

## 2. نطاق التنفيذ

تم تعديل النصوص الظاهرة فقط.

لم يتم تغيير:

- أي `id`.
- أي `templateId`.
- أي route.
- أي API.
- أي renderer.
- `services/audioEngine.ts`.
- Stream Deck logic.
- Player Intel V2 logic.
- روابط OBS أو آلية `Program OBS`.
- نظام IN/OUT نفسه.

## 3. ما تم تعريبه

### غرفة التحكم `Operator`

| قبل | بعد |
|---|---|
| `LIVE` | مباشر |
| `LIVE فقط` | المباشر فقط |
| `TAKE IN` | إدخال |
| `TAKE OUT` | إخراج |
| `LIVE ON AIR` | على الهواء |
| `OFF AIR` | خارج البث |
| `Single Program` | برنامج واحد |
| `Program OBS` | رابط البرنامج |
| `Copied` | تم النسخ |
| `SINGLE / MULTI` | فردي / متعدد |

كما تم تعريب tooltips الخاصة بإدخال/إخراج القالب ورابط البرنامج.

### المكتبة `Library`

تم تغيير:

- `Program OBS` إلى `رابط البرنامج`.
- رسالة النسخ إلى `تم نسخ رابط البرنامج`.
- tooltip الرابط العام ليصف أنه رابط البرنامج العام لكل القوالب الحية.

### المحرر `Editor`

تم تغيير:

- `ON AIR` إلى `على الهواء`.
- `Chroma` إلى `كروما`.
- `Edit Link` إلى `رابط التعديل`.
- `Copied` إلى `تم النسخ`.
- زر فتح رابط القالب أصبح يحمل نصًا واضحًا: `فتح الرابط`.

### الصفحة الرئيسية `Home`

تم تغيير:

- `ON AIR` إلى `على الهواء`.
- `Outputs` إلى `المخارج`.
- `Latency` إلى `الاستجابة`.
- `Quick Launch` إلى `قوالب سريعة`.
- `Live Match Stats` إلى `إحصائيات المباراة المباشرة`.
- `Lower Third` إلى `الشريط السفلي`.
- `Barcelona 2026` إلى `برشلونة 2026`.
- `Top Donors` داخل الوصف إلى `أبرز الداعمين`.

### صفحة الربط الخارجي `Integrations`

داخل سكربت إعداد أوامر Stream Deck تم تعريب labels المعروضة:

- `General Controls` إلى `أوامر عامة`.
- `Set ON AIR` إلى `إدخال للبث`.
- `Set OFF AIR` إلى `إخراج من البث`.
- `Toggle Visibility (legacy)` إلى `تبديل الظهور (قديم)`.
- `Scoreboard Controls` إلى `أوامر لوحة النتيجة`.

القيم الداخلية للأوامر مثل `set_on` و`set_off` لم تتغير.

## 4. ما تركته عمدا

تم ترك:

- أسماء المتغيرات والدوال مثل `programObsCopied`.
- تعليقات تقنية مثل `Browser Source`.
- قيم داخل `components/renderers/OverlayConstants.ts` لأنها تقع داخل renderers/constants وتحتاج مرحلة منفصلة حتى لا نخلط نطاق هذه الصيانة.
- كلمات تقنية مثل `Stream Deck` لأنها اسم منتج.

## 5. التحقق

تم تشغيل:

```bash
npm run lint
npm run build
```

النتيجة:

- `npm run lint`: ناجح.
- `npm run build`: ناجح.
- تحذير Vite الخاص بحجم chunk بقي موجودا، وهو تحذير قديم.

تم فحص عدد functions:

```text
functions=10
```

لا توجد endpoints جديدة.

## 6. الاختبار البصري

لم أدّعِ اختبارًا بصريًا لهذه المرحلة بسبب بوابة الترخيص المحلية التي تمنع فتح الواجهة مباشرة في المتصفح الداخلي بنفس مسار Vite.

الاختبار المؤكد:

- TypeScript compile.
- build إنتاجي.
- diff نصي فقط في صفحات الواجهة.
- عدم تغيير runtime أو الروابط.

## 7. QA يدوي مقترح

بعد فتح الواجهة بترخيص صالح:

1. افتح غرفة التحكم.
2. تأكد أن أزرار الإدخال والإخراج تظهر بالعربي.
3. انسخ رابط البرنامج وتأكد أن الزر يعرض `تم النسخ`.
4. افتح المحرر وتأكد أن زر رابط التعديل عربي.
5. افتح الصفحة الرئيسية وتأكد أن حالة النظام عربية.
6. افتح صفحة الربط الخارجي وتأكد أن أوامر Stream Deck المعروضة عربية.
7. تأكد أن `Program OBS` الوظيفي ما زال يعمل بنفس الرابط والمنطق.

## 8. تقييم الخطر

الخطر منخفض جدا:

- تغييرات نصية فقط.
- لا تغيير منطق.
- لا تغيير routes.
- لا تغيير API.
- لا تغيير audio.
- لا تغيير renderers.

## 9. التوصية التالية

الخطوة التالية الأنسب:

1. مرحلة صيانة صغيرة لعرض حالة الرابط الخاص داخل بطاقة القالب بشكل أوضح.
2. أو الانتقال إلى Mercato visual polish قالبا قالبا.

لا أنصح بفتح حماية token للروابط إلا كمرحلة منفصلة لأنها أعمق من مجرد UX.
