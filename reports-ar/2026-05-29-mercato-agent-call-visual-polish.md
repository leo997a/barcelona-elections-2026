# تقرير عربي: ترقية بصرية لقالب ميركاتو — غرفة اتصال الوكيل

التاريخ: 2026-05-29  
اسم المرحلة: MERCATO-AGENT-CALL-VISUAL-POLISH-001  
نوع المرحلة: Visual polish محدود لقالب ميركاتو واحد

## 1. الهدف

بعد تحسين غرفة التحكم وتجربة اختيار القوالب، بدأت هذه المرحلة مرورًا بصريًا مركزًا على قالب:

```text
template-mercato-x5-agent-call
```

الهدف هو رفع إحساس القالب من "محادثة نصية داخل بطاقة" إلى "غرفة اتصال ميركاتو قابلة للبث"، بدون تغيير `templateId`، وبدون تغيير حقول القالب، وبدون المساس بروابط OBS أو الصوت أو الـ API.

## 2. التشخيص قبل التعديل

القالب كان يعمل تقنيًا، ويعرض:

- اسم المتصل ودوره.
- حالة المكالمة.
- مدة الاتصال.
- بيانات الصفقة.
- سجل الرسائل بين المراسل والوكيل.
- مصدر المعلومة وتقدم الصفقة.

لكن بصريًا كانت هناك مشاكل واضحة:

- رأس القالب لم يكن يعطي إحساس "غرفة اتصال" كافية.
- حالة المكالمة كانت إنجليزية داخل قالب عربي.
- لوحة الصفقة لم تكن تعطي قراءة سريعة لقوة التفاوض.
- لم يكن هناك تلخيص لآخر إشارة مهمة من المكالمة.
- سجل المحادثة كان صحيحًا، لكنه يحتاج هوية بث أكثر وضوحًا.
- توزيع الأعمدة كان ضيقًا قليلًا للقالب عند عرضه في output.

## 3. ما تم تنفيذه

### 3.1 تعريب مؤشرات حالة المكالمة

تم تحويل الحالات المرئية من:

```text
LIVE / RECORDED / PRIVATE SOURCE
```

إلى:

```text
مباشرة / مسجلة / مصدر خاص
```

هذا يحافظ على قيم الحقول الداخلية كما هي، ويغير النص المعروض فقط.

### 3.2 رأس جديد لغرفة الاتصال

تمت ترقية الشريط العلوي إلى شبكة بث بثلاث مناطق:

- هوية المتصل والمدة وحالة الاتصال.
- عنوان مركزي: `غرفة اتصال الصفقة`.
- لوحة المراسل ونسبة الثقة.

كما تمت إضافة خط علوي ديناميكي يربط لون الحالة المباشرة بلون مرحلة التفاوض.

### 3.3 مرحلة تفاوض محسوبة

بدون إضافة أي حقل جديد، أصبح القالب يحسب مرحلة الصفقة من `confidencePct`:

- `إغلاق وشيك`
- `حسم متقدم`
- `تفاوض نشط`
- `رصد أولي`

كل مرحلة لها لون مختلف داخل الواجهة.

### 3.4 لوحة صفقة أكثر وضوحًا

تم تحسين لوحة الصفقة لتعرض:

- اسم اللاعب وصورته.
- النادي القادم منه والنادي المتوقع.
- قيمة الصفقة.
- نسبة الثقة.
- عدد رسائل المراسل.
- عدد رسائل الوكيل.

هذه الإضافة تعطي المخرج قراءة فورية لكثافة المكالمة دون البحث داخل الرسائل.

### 3.5 قناة محادثة مشفرة

تمت إضافة هوية أوضح لسجل المكالمة:

- أيقونة قفل.
- تسمية `قناة مشفرة · transcript`.
- waveform صغير عند الحالة المباشرة.

لم يتم تغيير منطق عرض الرسائل أو اتجاه النص، فقط تحسين شكل الحاوية.

### 3.6 آخر إشارة من المكالمة

تمت إضافة صندوق `آخر إشارة` في لوحة المصدر، يعرض آخر رسالة من `chatLines` مع احترام اتجاه العربية أو الإنجليزية.

هذا يساعد في البث لأن آخر سطر غالبًا هو خلاصة التطور الأحدث في الصفقة.

## 6. الاختبار البصري

تم تشغيل سيرفر Vite محلي منفصل على:

```text
http://127.0.0.1:5178
```

ثم تم فتح output embedded لقالب `agent_call` عبر Chrome headless والتقاط screenshot محلي للتحقق من الرندر.

ملاحظة مهمة: أول screenshot خرج أبيض، لذلك تم إعادة الفحص بتشخيص إضافي. الفحص الثاني أظهر القالب بشكل صحيح داخل المتصفح:

- العنوان المركزي `غرفة اتصال الصفقة` ظهر.
- لوحة الصفقة ظهرت مع اللاعب والقيمة والانتقال.
- الرسائل ظهرت داخل transcript.
- لوحة `آخر إشارة` ظهرت في جانب المصدر.
- لم تظهر صناديق صور مكسورة أو crash واضح.

ملفات screenshot وملفات Chrome profile كانت مؤقتة وتم حذفها قبل نهاية المرحلة، ولم تتم إضافتها إلى git.

## 9. قائمة QA يدوية مقترحة

1. افتح قالب Agent Call من Library.
2. غيّر حالة المكالمة بين live / recorded / private_source.
3. غيّر `confidencePct` إلى 30 و70 و85 و95 وتأكد من تغير مرحلة التفاوض.
4. أضف رسائل عربية وإنجليزية داخل `chatLines`.
5. تأكد أن `آخر إشارة` تعرض آخر رسالة فعلًا.
6. افتح رابط output الخاص بالقالب في OBS وتأكد أن الرابط لم يتغير.
7. تأكد أن الصوت لم يتغير، لأن هذه المرحلة لم تعدل النظام الصوتي.

## 10. الخلاصة

هذه المرحلة رفعت جودة قالب اتصال الوكيل بدون توسيع السطح التقني. القالب أصبح أقرب إلى غرفة بث ميركاتو حقيقية: عنوان واضح، حالة عربية، قراءة تفاوضية، لوحة صفقة، وسجل اتصال منظم.

الخطوة المنطقية التالية ليست refactor عام، بل الاستمرار قالبًا قالبًا داخل ميركاتو بنفس الأسلوب: تعديل محدود، اختبار بصري، تقرير عربي، ثم commit منفصل.
