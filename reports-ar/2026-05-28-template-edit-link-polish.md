# تقرير مرحلة: تحسين تجربة روابط تعديل القوالب

**التاريخ:** 2026-05-28  
**اسم المرحلة:** TEMPLATE-EDIT-LINK-POLISH-002  
**نوع المرحلة:** UX Hotfix / Navigation polish  
**النطاق:** تحسين تجربة رابط تعديل القالب بعد إضافة deep links، بدون endpoint جديد وبدون تغيير template IDs.

---

## 1. سبب المرحلة

بعد إضافة رابط تعديل ثابت لكل قالب بصيغة:

```text
/Library#/edit/{overlayId}
```

بقيت فجوتان عمليتان:

1. المستخدم يستطيع نسخ الرابط من بطاقة القالب في Library، لكن داخل Editor نفسه لا يوجد زر واضح لنسخ رابط القالب المفتوح.
2. إذا فتح المستخدم رابطًا لقالب محذوف أو غير موجود في هذا الاستوديو، كان التطبيق يعود إلى Library بصمت، وهذا قد يبدو وكأنه عطل.

هذه المرحلة تكمل التجربة بدون توسيع معماري.

---

## 2. التشخيص

تم فحص:

- `App.tsx`
  - يدعم `#/edit/{overlayId}`.
  - يفتح الـ Editor إذا وُجد القالب.
  - كان لا يرسل أي إشارة للـ Library عند فشل العثور على القالب.

- `pages/Editor.tsx`
  - يحتوي شريطًا علويًا في منطقة المعاينة.
  - يحتوي بالفعل زر OBS/Monitor و Chroma.
  - المكان مناسب لإضافة زر Copy/Edit Link بدون تعديل بنية التبويبات أو حقول القالب.

- `pages/Library.tsx`
  - يدعم زر نسخ رابط تعديل داخل بطاقة القالب.
  - كان يحتاج فقط prop اختياري لعرض حالة الرابط المفقود.

---

## 3. ما تم تنفيذه

### 3.1 زر `Edit Link` داخل Editor

تمت إضافة زر صغير في الشريط العلوي للمعاينة داخل `pages/Editor.tsx`:

```text
Edit Link
```

الزر:

- يستخدم `syncManager.buildEditUrl(liveOverlay.id)`.
- ينسخ رابط تعديل القالب الحالي.
- يعرض feedback قصيرًا: `Copied`.

هذا يجعل الرابط متاحًا أثناء العمل على القالب مباشرة، وليس فقط من Library.

### 3.2 حالة رابط مفقود

في `App.tsx` تمت إضافة تتبع تحميل overlays:

```ts
hasLoadedOverlays
```

ثم يتم حساب:

```ts
missingEditOverlayId
```

فقط بعد تحميل الحالة المحلية، حتى لا تظهر رسالة خطأ مؤقتة أثناء أول تحميل.

### 3.3 رسالة واضحة داخل Library

في `pages/Library.tsx` تمت إضافة banner صغير عندما يكون الرابط يشير إلى قالب غير موجود:

```text
رابط القالب غير موجود في هذا الاستوديو.
```

مع عرض الـ `overlayId` المفقود بصيغة مختصرة.

## 7. المخاطر المتبقية

1. **الرابط ما زال محليًا**
   - يعمل للمستخدم/الاستوديو الذي يملك القالب في التخزين المحلي.
   - لا يشارك القالب مع مستخدم آخر.

2. **لا توجد صفحة "استعادة قالب"**
   - إذا حذف المستخدم القالب ثم فتح الرابط، سيظهر التنبيه فقط.
   - يمكن لاحقًا إضافة import/export أو سلة محذوفات، لكن ليس في هذه المرحلة.

3. **زر النسخ يعتمد على Clipboard API**
   - في المتصفحات الحديثة يعمل طبيعيًا.
   - إذا رفض المتصفح clipboard permission، يظهر alert بدل الفشل الصامت.

## 8. التوصية التالية

الأولوية التالية المنطقية:

1. تثبيت سياسة `Program OBS`: هل نسمح بتعدد القوالب LIVE أم نضيف خيار "Single Program Mode" يخرج السابق تلقائيًا؟
2. بعد ذلك يمكن إضافة مؤشر أو إعداد صغير في Operator يوضح حالة Program Output.
3. تأجيل أي sharing cloud أو حسابات قوالب حتى يتم تثبيت UX المحلي بالكامل.

هذه المرحلة لم تغيّر runtime ولا audio ولا Player Intel، وهي آمنة ومحصورة في تجربة التنقل والنسخ.
