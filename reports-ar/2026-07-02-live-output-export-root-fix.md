# تقرير إصلاح جذري لمسار الظهور والتصدير

التاريخ: 2026-07-02

## المشكلة

ظهرت ثلاث مشاكل متداخلة:

- زر الظهور والإخفاء كان يعتمد على نشر واحد سريع، وإذا فشل طلب `/api/live` أو تأخر لم يكن هناك ضمان كاف لإعادة الإرسال.
- رابط العرض العام كان ينتظر قناة البث المباشر قبل جلب الحالة النهائية، وهذا كان يسمح لحالة قديمة أو fallback بأن تبقى ظاهرة للمستخدم.
- تصدير الصورة كان يفشل مع بعض الصور الخارجية المحمية لأن المتصفح يمنع تحويلها إلى canvas صالحة للتصدير.

## الحل المنفذ

- تقوية نشر أوامر الظهور والإخفاء بإعادة محاولات متدرجة وفحص صريح لفشل HTTP.
- جعل رابط العرض العام يجلب الحالة الرسمية من `/api/live` فور فتح الرابط قبل الاعتماد على قناة البث.
- إضافة وكيل صور آمن `/api/image-proxy` لاستخدامه عند فشل تحميل الصور الخارجية مباشرة داخل التصدير.
- منع حقول الألوان من استلام قيمة فارغة حتى لا تظهر تحذيرات المتصفح ولا تنكسر إعدادات الثيم.

## التحقق

- `npm run lint`: ناجح.
- `node --test tests/mondial-runtime-controls.test.mjs tests/template-image-export.test.mjs`: ناجح، 20 اختبار من 20.
- `npm run build`: ناجح.
- اختبار إنتاج محلي لمسار `/api/live`: مخفي -> ظاهر -> مخفي، ونجح بنفس المعرف دون تغيير الرابط.

## النتيجة

المشكلة لم تعد معالجة كزر منفرد فقط. تم تقوية مسار التحكم كاملا: المحرر، الرابط العام، التخزين الحي، إعادة المحاولة، والتصدير.
