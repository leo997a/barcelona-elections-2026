# تقرير مرحلة: تعريب قوالب Mercato Media + رفع محلي + ثيمات متعددة

التاريخ: 2026-05-28  
اسم المرحلة: MERCATO-MEDIA-ARABIC-UPLOAD-THEMES-001  
نوع المرحلة: تصحيح UX وتوسيع آمن داخل القوالب الجديدة بدون API جديد

## 1. الملخص التنفيذي

تم تنفيذ مرحلة صغيرة ومحدودة لقوالب `Mercato Media Story` الجديدة بعد طلب جعل القوالب عربية، ودعم رفع الصور والفيديوهات محليًا، وإضافة إعدادات أدق وثيمات متعددة لكل قالب.

النتيجة الحالية:

- قوالب Mercato Media العشرة أصبحت عربية في الاسم، الوصف، العناوين الافتراضية، نقاط القصة، وواجهة العرض داخل renderer.
- كل قالب أصبح يملك قائمة ثيمات متعددة غير مكررة بدل الاعتماد على لون ثابت واحد.
- تمت إضافة إعدادات دقيقة للتحكم في:
  - ثيم القالب.
  - قوة تعتيم الميديا.
  - نعومة/blur الخلفية.
  - سطوع الصورة أو الفيديو.
  - شفافية اللوحات.
  - حجم النص.
- تمت إضافة رفع محلي ناجح داخل Editor لحقول الميديا الجديدة `mediaUrl` و`mediaAltUrl`، مع دعم الصور والفيديوهات القصيرة.
- تم حفظ الملف المحلي كـ `data:` URL داخل القالب حتى ينتقل مع رابط العرض/OBS دون الحاجة إلى رفع خارجي.
- لم يتم إضافة endpoints جديدة ولم يتغير عدد Vercel functions.

## 2. ما تم تغييره

### Editor

تم تحديث `pages/Editor.tsx` لإضافة:

- زر رفع صورة/فيديو داخل حقول الميديا الجديدة.
- قبول `image/*,video/*` فقط عندما يكون الحقل مخصصًا للميديا.
- إبقاء حقول الصور القديمة على `image/*` فقط.
- حد أمان 12MB للملفات المحلية داخل state.
- ضبط `mediaMode` تلقائيًا إلى `image` أو `video` عند رفع الملف الرئيسي.
- نقل حقول الميديا إلى تبويب الصور بدل ظهورها كحقول نصية خام.
- نقل حقول الموسيقى إلى تبويب الصوت.
- نقل إعدادات الثيم والتحكم البصري إلى تبويب التنسيق.

### Constants

تم تحديث `constants.ts` لإضافة:

- قائمة `MERCATO_MEDIA_THEMES` بسبعة ثيمات:
  - استوديو سماوي زجاجي.
  - ذهبي ملكي فاخر.
  - برتقالي اليوم الأخير.
  - أخضر الفحص الطبي.
  - أحمر مصادر عاجلة.
  - بنفسجي سوشيال.
  - أزرق منتصف الليل.
- تعريب أسماء ومسارات محتوى قوالب Mercato Media العشرة.
- تعريب labels الخاصة بالميديا والصوت والنداء والتحكم البصري.
- إضافة حقول إعدادات دقيقة لكل قالب.
- إبقاء IDs كما هي بدون تغيير.

### Renderer

تم تحديث `components/renderers/MercatoMediaStoryRenderer.tsx` لإضافة:

- تعريب الكلمات hardcoded داخل التصميم مثل المصدر، نسبة الحسم، الساعة، خط التسجيل، فتح الخزنة.
- دعم `data:image` و`data:video` في auto-detect.
- تطبيق الثيم المختار فعليًا على الألوان والخلفية.
- تطبيق إعدادات التعتيم والسطوع والـ blur وحجم النص على العرض.
- إبقاء الموسيقى الحقيقية تعمل كما كانت، مع مؤشر عربي لحالة الموسيقى.

### Template Registry

تم تحديث وصف `MERCATO_MEDIA_STORY` في `utils/templateRegistry.ts` ليكون عربيًا ويذكر دعم الميديا المحلية والرابط المباشر.

## 3. القيود المهمة

الرفع المحلي هنا مناسب للصور والفيديوهات القصيرة فقط، لأن الملف يتحول إلى `data:` URL داخل state. هذا ممتاز لسرعة OBS والرابط الخاص بالقالب، لكنه ليس مناسبًا للفيديوهات الكبيرة أو الطويلة.

التوصية للفيديوهات الكبيرة:

- وضعها داخل `public/` واستخدام رابط داخلي.
- أو استخدام رابط مباشر خارجي ثابت.
- أو بناء مرحلة لاحقة لإدارة media library حقيقية مع تخزين منفصل، وهذا يحتاج تصميم معماري مستقل.

## 6. المتبقي المقترح

لتحقيق عبارة "كل القوالب بالعربي" على مستوى المشروع كاملًا، توجد مرحلة لاحقة مقترحة:

1. تعريب قوالب Mercato legacy داخل `TRANSFER_NEWS` بالكامل.
2. تعريب labels القديمة مثل `Main headline`, `Source`, `Scale`, `Look variant`.
3. تعريب قيم JSON الافتراضية القديمة أو تحويلها إلى placeholders عربية غير مضللة.
4. مراجعة renderers القديمة للتأكد من عدم وجود كلمات hardcoded بالإنجليزية.
5. اختبار بصري لكل عائلة قوالب في المتصفح قبل push.

هذه المرحلة لم تنفذ ذلك عمدًا حتى لا يتم تعديل مساحة كبيرة من القوالب القديمة دفعة واحدة.
