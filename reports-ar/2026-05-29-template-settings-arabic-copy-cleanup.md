# تقرير صيانة: تنظيف نصوص إعدادات القوالب والنسخ الافتراضية

التاريخ: 2026-05-29  
اسم المرحلة: TEMPLATE-SETTINGS-ARABIC-COPY-CLEANUP-001  
نوع المرحلة: صيانة نصية محدودة  
الملفات المعدلة: `constants.ts` فقط + هذا التقرير

## 1. الهدف

بعد تنظيف أسماء القوالب الظاهرة في المرحلة السابقة، بقيت داخل إعدادات بعض القوالب نصوص إنجليزية تظهر للمستخدم عند فتح الإعدادات أو عند تشغيل القالب لأول مرة.

هذه المرحلة ركزت على تنظيف هذه البقايا بدون لمس بنية القوالب أو أي ربط تشغيلي.

## 2. نطاق التنفيذ

تم تعديل:

- labels ظاهرة داخل إعدادات القوالب.
- نصوص افتراضية تظهر في القالب مثل عناوين العدّاد والتنبيهات.
- بعض option labels المعروضة للمستخدم.
- JSON افتراضي لإحصائيات اللاعب حتى يظهر بالعربي عند أول فتح.

لم يتم تعديل:

- `id`
- `templateId`
- `type`
- قيم select التشغيلية مثل `CLOUD_BRIDGE`, `SCOUT_CARD`, `WORLD_FEED`.
- renderers.
- API.
- الصوت.
- Stream Deck.
- Player Intel V2 logic.
- روابط OBS أو live runtime.

## 3. أهم ما تم تنظيفه

### قوالب انتخابات برشلونة

تم تحويل نصوص افتراضية كانت إنجليزية:

| قبل | بعد |
|---|---|
| Last update 20:45 | آخر تحديث 20:45 |
| Style | النمط |
| Other / Undecided | آخرون / غير محسوم |
| Closing Countdown | العد التنازلي للإغلاق |
| Days / Hours / Minutes / Seconds | أيام / ساعات / دقائق / ثوان |
| Breaking | عاجل |
| Election desk | مكتب الانتخابات |
| Key Statement | تصريح بارز |
| مؤثر TAKE IN | مؤثر الإدخال |
| مؤثر TAKE OUT | مؤثر الإخراج |

### قوالب كرة القدم العالمية

تم تنظيف النصوص الافتراضية داخل scorebug / match bar / VAR / lower third / ticker:

| قبل | بعد |
|---|---|
| WORLD MATCH LIVE | مباراة عالمية مباشرة |
| LIVE | مباشر |
| ELITE FOOTBALL NIGHT | ليلة كروية نخبوية |
| MATCHDAY LIVE | يوم المباراة مباشر |
| 2nd Half | الشوط الثاني |
| HT | استراحة بين الشوطين |
| CHECKING POSSIBLE PENALTY | مراجعة ركلة جزاء محتملة |
| TACTICAL DESK | طاولة التحليل |
| MATCH CENTER | مركز المباراة |
| MATCH UPDATE | تحديث المباراة |

### Player Stats Lab

تم تنظيف labels كانت تظهر للمستخدم:

| قبل | بعد |
|---|---|
| REO Player Cloud Bridge | جسر REO السحابي للاعبين |
| Scout Card / قائمة مراقبة 3 لاعبين | بطاقة كشاف / قائمة مراقبة 3 لاعبين |
| Selected metric keys | مفاتيح الإحصائيات المختارة |
| Hero metric keys | مفاتيح الإحصائيات الرئيسية |
| Secondary metric keys | مفاتيح الإحصائيات الثانوية |
| Hidden metric keys | مفاتيح الإحصائيات المخفية |
| Natural metric search | بحث الإحصائيات الطبيعي |
| English | إنجليزي |

كما تم تحويل labels اللاعبين من A/B/C إلى:

- اسم اللاعب الأول.
- اسم اللاعب الثاني.
- اسم اللاعب الثالث.

### JSON الإحصائيات الافتراضي

تم تحويل labels/hints داخل `playerStatsJson` من الإنجليزية إلى العربية:

- Goals -> الأهداف.
- Shots / 90 -> التسديدات / 90.
- Key passes -> تمريرات مفتاحية.
- Progressive passes -> تمريرات تقدمية.
- Recoveries -> استرجاع الكرة.
- Minutes -> الدقائق.

تم ترك `category` كما هي لأنها قيمة تصنيف داخلية.

## 4. لماذا لم أترجم كل شيء؟

تم ترك بعض القيم كما هي عمدا:

- أسماء لاعبين مثل `Robert Lewandowski` و`Cole Palmer`.
- أسماء أندية داخل حقول البحث مثل `Barcelona` و`Chelsea`.
- اختصارات الفرق مثل `BAR`, `MAD`, `HIL`, `NAS`.
- قيم select الداخلية مثل `WORLD_FEED`, `MATCH_NIGHT`, `CLOUD_BRIDGE`.
- مصطلحات تقنية مثل `JSON`, `API`, `FBref`, `WhoScored`.

السبب: هذه القيم قد تُستخدم كمدخلات بحث أو مفاتيح تشغيل أو محتوى قابل للتعديل، وترجمتها عشوائيا قد تضعف الربط مع الجسور والـ renderers.

## 5. التحقق

تم تشغيل:

```bash

```

## 7. الاختبار البصري

لم أعدّ الاختبار البصري داخل المتصفح في هذه المرحلة، لأن المتصفح الداخلي يتوقف عند بوابة الترخيص المحلية، وخادم Vite على `5177` لا يخدم `/api/license` محليا بنفس طريقة الإنتاج.

لذلك لا أدعي اختبارا بصريا لهذه المرحلة. الاختبار المؤكد هو TypeScript/build/diff/functions.

## 8. QA يدوي مقترح

بعد فتح بيئة بترخيص صالح:

1. افتح Editor لقالب انتخابات برشلونة وتأكد أن العدّاد والعناوين الافتراضية عربية.
2. افتح scorebug كروي عالمي وتأكد أن الحالة تظهر `مباشر` و`الشوط الثاني`.
3. افتح قالب VAR وتأكد أن النص الفرعي عربي.
4. افتح Player Stats Lab وتأكد أن خيارات الجسر والبطاقة واللاعب الأول/الثاني/الثالث عربية.
5. تأكد أن TAKE IN/OUT يعمل كما كان لأن IDs لم تتغير.
6. افتح الرابط العام في OBS أو المتصفح وتأكد أن الاسم والرابط لم يتأثرا.

## 9. تقييم الخطر

الخطر منخفض:

- التعديل نصي.
- لا توجد endpoints جديدة.
- لا يوجد لمس للصوت أو الرندر أو Stream Deck.
- لا توجد ملفات صوتية أو ميديا خارجية.
- لا يوجد تغيير على secrets أو `.env`.

الخطوة التالية الأفضل هي واحدة من مسارين:

1. صيانة صغيرة لواجهة إنشاء القالب والرابط الخاص لكل مستخدم.
