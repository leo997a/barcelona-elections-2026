# تقرير صيانة: تنظيف أسماء القوالب الظاهرة بالعربي

التاريخ: 2026-05-29  
اسم المرحلة: TEMPLATE-VISIBLE-ARABIC-NAMES-CLEANUP-001  
نوع المرحلة: صيانة واجهة صغيرة وآمنة  
الملفات المعدلة: `constants.ts` فقط + هذا التقرير

## 1. الهدف

كان الهدف من هذه المرحلة إغلاق جزء مزعج من تجربة المكتبة وغرفة التحكم: وجود أسماء قوالب ظاهرة للمستخدم بصياغة مختلطة عربي/إنجليزي، مثل:

- `Projection Football - Lineup Board`
- `Player Stats Lab - بطاقة إحصائيات لاعب`
- `كروي عالمي - Scorebug علوي`
- `خبر حصري (Exclusive Alert)`
- `H2H مقارنة لاعبين`

هذا النوع من الخلط لا يكسر الرندر، لكنه يجعل غرفة التحكم تبدو كأنها غير مصقولة، خصوصا بعد طلب أن تكون القوالب عربية وأكثر احترافية.

## 2. نطاق التنفيذ

تم الالتزام بنطاق ضيق:

- تعديل أسماء القوالب الظاهرة فقط.
- تعديل أوصاف قليلة تظهر في بطاقات المكتبة.
- تعريب بعض labels الآمنة داخل إعدادات البروجكشن وتحليل المباراة.
- عدم تغيير أي `id`.
- عدم تغيير أي `templateId`.
- عدم تغيير أي `type`.
- عدم تغيير أي قيمة تشغيلية يستخدمها renderer مثل `designStyle` أو `themePreset`.
- عدم لمس `components/renderers`.
- عدم لمس `services/audioEngine.ts`.
- عدم لمس `api`.
- عدم لمس Player Intel V2 logic.
- عدم لمس Stream Deck.
- عدم لمس روابط OBS أو آلية العرض المباشر.

## 4. إعدادات تم تعريبها بدون تغيير السلوك

تم تعريب labels في قالب البروجكشن لأنها تظهر للمستخدم داخل الإعدادات:

- `Projection style` أصبح `نمط البروجكشن`.
- `Projection theme` أصبح `ثيم البروجكشن`.
- `Main title` أصبح `العنوان الرئيسي`.
- `Team logo` أصبح `شعار الفريق`.
- `Players shown` أصبح `عدد اللاعبين الظاهرين`.
- `Template scale` أصبح `حجم القالب`.
- حقول اللاعبين أصبحت `رقم اللاعب` و`اسم اللاعب` و`صورة اللاعب`.

كما تم تعريب بعض labels/option labels الآمنة:

- خيارات المشهد الصوتي في البداية: `تأكيد الصفقة` و`الاتفاق قريب`.
- مصدر بيانات المباراة: `جسر REO السحابي — Google Cloud` و`جسر محلي — localhost:3005`.
- أنماط عرض إحصائيات المباراة: `سكتان مزدوجتان`، `تقسيم تكتيكي`، `برج البيانات`، `بث مدمج`، `استوديو زجاجي`، `خط تماس نيون`.
- جودة البث: `بث فائق` و`قياسي`.

القيم الداخلية مثل `DUAL_RAIL` و`MATCH_NIGHT` و`TITLE_STING` بقيت كما هي حتى لا ينكسر الرندر.

## 5. ما لم يتم تغييره

بقيت بعض الكلمات الإنجليزية عمدا لأنها ليست أسماء قوالب أو لأنها قيم تقنية:

- أسماء الصحفيين والمصادر داخل JSON مثل `Fabrizio Romano` و`David Ornstein`.
- أسماء خدمات أو صيغ تقنية مثل `JSON`, `API`, `FBref`, `FotMob`, `Google Cloud`.
- قيم select الداخلية التي يعتمد عليها الرندر.
- أسماء أندية أو لاعبين داخل بيانات افتراضية مثل Barcelona وMadrid لأنها محتوى قابل للتعديل من المستخدم وليست label نظام.

## 6. نتيجة التدقيق بعد التعديل

تم فحص أسماء القوالب داخل `constants.ts` عبر:

```bash
rg -n "name: '.*[A-Za-z].*'" constants.ts
```

النتيجة بعد التنظيف:

- لم يبق أي اسم قالب ظاهر مختلط إنجليزي/عربي.
- المتبقي فقط أسماء مصادر داخل بيانات JSON في قوالب الميركاتو.

## 7. التحقق

تم تشغيل:

```bash
npm run lint
npm run build
```

النتيجة:

- `npm run lint`: ناجح.
- `npm run build`: ناجح.
- تحذير Vite الخاص بحجم chunk أكبر من 500 kB ما زال موجودا، وهو تحذير معروف وليس ناتجا عن هذه المرحلة.

تم فحص عدد functions:

```bash
functions=10
```

العدد بقي أقل من الحد المطلوب 12.

## 8. ملاحظة عن الاختبار البصري

تم فتح `/Library` محليا على:

```text

```

لكن المتصفح الداخلي وصل إلى بوابة الترخيص. حاولت استخدام مفتاح QA مؤقت، لكن خادم Vite المحلي لا يخدم `/api/license` على نفس المنفذ، لذلك لم أستطع الدخول للمكتبة بصريا من المتصفح الداخلي في هذه المرحلة.

لذلك لا أدعي أنني اختبرت الشاشة بصريا بعد هذا التعديل. الاختبار المؤكد هنا هو:

- فحص النصوص داخل مصدر الحقيقة `constants.ts`.
- نجاح TypeScript lint.
- نجاح build الإنتاجي.
- ثبات عدد functions.

## 9. قائمة QA اليدوية المقترحة

بعد نشر التغيير أو فتح بيئة لديها ترخيص صالح:

1. افتح صفحة المكتبة.
2. ابحث عن `Projection` وتأكد أن بطاقات البروجكشن تظهر بأسماء عربية.
3. ابحث عن `Scorebug` وتأكد أن القالب يظهر باسم `كروي عالمي — شريط النتيجة العلوي`.
4. افتح غرفة التحكم وتأكد أن قائمة التشغيل تعرض الأسماء الجديدة.
5. أضف قالبا من المكتبة وتأكد أن الرابط المباشر وOBS لا يتغيران بسبب الاسم.
6. افتح Editor لقالب بروجكشن وتأكد أن labels الأساسية عربية.
7. جرّب TAKE IN/OUT لقالب موجود للتأكد أن `templateId` لم يتأثر.

## 10. تقييم الخطر

الخطر منخفض، لأن التغيير نصي في `constants.ts`:

- لم تتغير IDs.
- لم تتغير types.
- لم تتغير renderers.
- لم تتغير APIs.
- لم يتغير audio runtime.
- لم يتغير Stream Deck.
- لم يتغير Player Intel V2 logic.

الخطر الوحيد الممكن هو أن يكون لدى المستخدمين قوالب محفوظة بأسماء قديمة ويرون الاسم الجديد في القوالب الجديدة فقط حسب طريقة حفظ النسخ. هذا مقبول، لأن الاسم الظاهر قابل للتحرير لاحقا في غرفة التحكم حسب المرحلة السابقة.

## 11. توصية المرحلة التالية

الأولوية التالية المقترحة:

1. مرحلة صغيرة لتنظيف labels المتبقية في إعدادات القوالب القديمة بدون تغيير القيم التشغيلية.
2. مرحلة منفصلة لتحسين UX إنشاء القالب وتثبيت الرابط الخاص بالمستخدم.
3. مرحلة منفصلة للميديا المحلية والصوت الحقيقي، لأنها تحتاج تصميم تخزين/رفع آمن ولا يجب خلطها مع تنظيف النصوص.
4. مرحلة منفصلة لمرئيات الميركاتو قالبا قالبا.

هذه المرحلة لا تبدأ أيا من تلك المراحل؛ هي فقط تغلق جزءا من فوضى الأسماء الظاهرة.
