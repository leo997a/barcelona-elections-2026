# REO Data Sync Tray - دليل الاستخدام

## ما هو REO Data Sync Tray؟

تطبيق خفيف يظهر كايقونة صغيرة بجانب الساعة في Windows.
يقوم بجلب بيانات FBref يوميا من الجهاز المحلي ورفعها الى VPS.

**لا يعمل 24 ساعة** - يشتغل مرة واحدة عند فتح الجهاز ثم يغلق نفسه.

---

## كيف يشتغل؟

1. عند تسجيل الدخول الى Windows، يبدا تلقائيا
2. يفحص هل تم التشغيل اليوم (من ملف state)
3. اذا تم التشغيل اليوم: يعرض اشعار ويغلق بعد 5 ثواني
4. اذا لم يتم التشغيل: يبدا جلب البيانات بالخلفية
5. يعرض حالة في tooltip الايقونة
6. عند الانتهاء يرسل Windows notification ثم يغلق

---

## كيف اعرف انه يعمل؟

ابحث عن ايقونة دائرية زرقاء بحرف R بجانب الساعة.
اذا مخفية، اضغط السهم الصغير بجانب الساعة لاظهار الايقونات المخفية.

الالوان:
- ازرق: جاهز / اكتمل
- ذهبي: يعمل الان
- احمر: فشل
- برتقالي: ملغي / مقفل

---

## القائمة (كليك يمين على الايقونة)

| الخيار | الوظيفة |
|--------|---------|
| Status | يعرض الحالة الحالية |
| Open Logs | يفتح ملف log اليوم في Notepad |
| Run Now | تشغيل يدوي (يتجاوز شرط مرة باليوم) |
| Cancel Current Run | الغاء التشغيل الحالي |
| Exit | اغلاق التطبيق |

---

## Run Now يدوي

اضغط كليك يمين على الايقونة واختر Run Now.
هذا يتجاوز شرط "مرة واحدة باليوم" لكن:
- لا يتجاوز lock file (اذا نسخة تانية شغالة)
- لا يرفع cache فاشل

---

## فتح Logs

اضغط Open Logs من القائمة.
او افتح الملف يدويا:

```
deploy\reo-datafootball-worker\.logs\reo-sync-YYYY-MM-DD.log
```

---

## ايقاف التطبيق

اضغط كليك يمين على الايقونة واختر Exit.
اذا كان sync يعمل، سيسألك قبل الاغلاق.

---

## حذف التشغيل التلقائي

شغل من PowerShell:

```powershell
cd deploy\reo-datafootball-worker\windows_tray
.\uninstall_startup_task.ps1
```

هذا يحذف Scheduled Task فقط. لا يحذف logs او cache.

---

## تثبيت التشغيل التلقائي

```powershell
cd deploy\reo-datafootball-worker\windows_tray
.\install_startup_task.ps1
```

---

## ماذا يحدث اذا FBref فشل؟

- يكتب الخطا في log
- لا يرفع cache فاشل الى VPS
- الكاش القديم على VPS يبقى محمي
- يعرض اشعار: "REO Data Sync failed"
- يغلق نفسه

---

## ماذا يحدث اذا الجهاز مغلق؟

- لا يحصل شيء
- عند فتح الجهاز، يحاول التشغيل
- اذا فات اكثر من يوم، يحاول مرة واحدة فقط

---

## كيف يحمي الكاش القديم؟

1. يجلب البيانات الى مجلد مؤقت
2. يتحقق من صحة البيانات (validate_cache.py)
3. اذا التحقق فشل: لا يرفع ولا يستبدل
4. اذا نجح: يضغط ويرفع عبر SCP
5. VPS يعمل backup للقديم قبل التثبيت

---

## ملفات المشروع

```
windows_tray/
  ReoDataSyncTray.ps1      - التطبيق الرئيسي
  install_startup_task.ps1  - تثبيت Task Scheduler
  uninstall_startup_task.ps1 - حذف Task Scheduler
  README_TRAY_AR.md         - هذا الملف

.state/
  reo-sync-state.json       - حالة آخر تشغيل
  reo-sync.lock             - قفل منع التشغيل المتكرر

.logs/
  reo-sync-YYYY-MM-DD.log   - log كل يوم
```
