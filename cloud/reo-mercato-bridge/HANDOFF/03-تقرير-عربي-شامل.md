# 03 — التقرير العربي الشامل: جسر الانتقالات الموحّد

تاريخ: 2026-06-29 · الحالة: مبني ومُثبت ويعمل ببيانات حقيقية.

## 1. الهدف
بناء مصدر بيانات انتقالات **حقيقي ودقيق** لقوالب الميركاتو في REO، يجمع:
- **FotMob** للخلاصة الحيّة والبصريات (صور لاعبين، شعارات أندية، حدث الصفقة).
- **Transfermarkt** للعمق المرجعي الدقيق (الرسوم الرسمية، القيمة السوقية، الاتجاه، المسيرة).

## 2. المشاكل التي عُولجت

| المشكلة السابقة | الحل المطبّق | الإثبات |
|-----------------|--------------|---------|
| البيانات وهمية/curated | اكتشاف نقطة FotMob الحقيقية + التقاطها بمتصفح | Anthony Gordon → Barcelona €80M (2026-05-29) حقيقي |
| بيانات الموسم الماضي | فلترة `date >= 2026-05-01` (موسم 2026/27) | 22 صفقة (الأغلى) + 48 (الأحدث) للموسم الحالي |
| لا صور/شعارات | روابط `images.fotmob.com` عبر `playerId`/`clubId` | تظهر في اللوحة فعلياً |
| لا زر تحديث | `/api/refresh` يلتقط حيّاً من FotMob | زر «🔄 تحديث» في اللوحة |
| الرسوم/القيمة غير دقيقة | الرسوم الرسمية + القيمة من Transfermarkt | Gordon: رسوم €80.0m + قيمة €75M من TM |

## 3. المعمارية (الدمج العبقري)
```
FotMob (متصفح CDP) ──► تطبيع + فلترة موسم ──┐
                                            ├──► دمج (mergeSources) ──► بطاقة REO موحّدة
Transfermarkt (مرجع) ──► بطاقة الذاكرة ──────┘        (كل حقل موسوم بمصدره)
```
- كل حقل `{ value, _src }` — شفافية كاملة عن المصدر.
- تدرّج آمن: غياب TM → رسوم/قيمة من FotMob؛ غياب FotMob → الكاش.

## 4. ما الذي أراه الآن بصرياً؟
- **`:4319/`** لوحة الدمج: بطاقات بصور وشعارات حقيقية، تبويبا (أغلى/أحدث)، زر تحديث، تفاصيل بمصادر ملوّنة.
- **`:4319/overview`** الصورة العامة: 32 قالباً مصنّفة + مصدر بيانات كلٍّ + روابط تحكّم/إخراج.
- **`:3001/`** معرض الميركاتو الأصلي: التصاميم الكاملة + لوحات التحكم.
- **`:4317/`** جسر Transfermarkt المرجعي.

## 5. خريطة تغذية القوالب الـ32
انظر `/overview` للتفصيل المرئي. ملخص:
- **13 قالب صفقات حيّة** ← الدمج الموحّد (FotMob+TM).
- **10 قوالب مالية** ← تجميع الرسوم حسب النادي/الدوري.
- **5 قوالب عدّاد/نافذة** ← تواريخ النافذة (15/5 → 1/9 2026).
- **2 مرجعي** (مقارنة لاعبين) ← Transfermarkt.
- **2 أخبار/يدوي** (شائعات، اقتباس مدرب).

## 6. المتبقّي لكوديكس (خطوات التنفيذ في الإنتاج)
1. نشر الجسر كخدمة Node+Chrome مستقلة في `cloud/`.
2. مستهلِك `api/transfers-live.ts` بسلّم fallback.
3. تحميل قاعدة Transfermarkt الحقيقية (CC0 datasets) بدل بذرة DEMO.
4. تغذية القوالب الـ32 من العقد الموحّد + زر تحديث في كل قالب.
5. **إصلاح الثيمات لتؤثّر على القالب المعروض** (لا الإعدادات فقط) — راجع تقرير الميركاتو `تقرير-الربط-بالأداة/`.
6. **جعل كل النصوص قابلة للتحرير** عبر حقول `OverlayField` (نمط REO).
7. اختبارات عقد + تحقّق إنتاج (لقطات).

## 7. ملاحظات قانونية
- FotMob: بيانات عبر المتصفح — احترم نافذة التحديث (5–15 د) والإسناد «المصدر: FotMob».
- Transfermarkt: الأساس من `transfermarkt-datasets` (CC0). الـ scraper الموجّه يُحسم ترخيصه قبل الإنتاج.

## 8. الخلاصة
الجسران مبنيان ومُثبتان ومدموجان ببيانات حقيقية 2026/27. الحزمة جاهزة للتسليم لكوديكس
مع توثيق كامل (نقاط FotMob، عقد الدمج، خطة الربط). البصريات والتدقيق موزّعان بذكاء بين المصدرين.
