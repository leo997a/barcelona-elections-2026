# 02 — عقد الدمج الموحّد (FotMob × Transfermarkt)

> «الحل العبقري»: كل مصدر يُستخدم فيما يتفوّق فيه، وكل حقل موسوم بمصدره.

## مصفوفة المصادر (مَن يغذّي ماذا)

| الحقل | المصدر الأساسي | التدرّج الآمن | السبب |
|------|----------------|----------------|-------|
| صورة اللاعب | **FotMob** (`playerId`) | — | FotMob يملك CDN صور موثوق. |
| شعار النادي | **FotMob** (`clubId`) | — | شعارات teamlogo مباشرة. |
| اللاعب/المركز/الأندية | **FotMob** | — | من الخلاصة الحيّة. |
| تاريخ الصفقة + «تمت اليوم» | **FotMob** (`transferDate`) | — | حدث لحظي. |
| النوع (دائم/إعارة/حر/تجديد) | **FotMob** (`onLoan`,`fee`) | — | من الخلاصة. |
| **الرسوم الرسمية** | **Transfermarkt** (سجل الانتقال) | FotMob `fee.value` | TM أدقّ للرسوم الرسمية. |
| **القيمة السوقية** | **Transfermarkt** (`currentValueEur`) | FotMob `marketValue` | TM مرجع القيمة. |
| اتجاه القيمة (Δ سنة) | **Transfermarkt** | — (يبقى غير متاح) | TM فقط يملك التاريخ. |
| تاريخ القيمة (مخطّط) | **Transfermarkt** | — | TM فقط. |
| تاريخ الانتقالات/آخر رسوم سابقة | **Transfermarkt** | — | TM فقط. |
| المشاركات/الأهداف/الدقائق | **Transfermarkt** | — | TM فقط. |
| مستوى الدوري | **Transfermarkt** | — | TM فقط. |
| العمر/الجنسية | **Transfermarkt** | — | TM دقيق. |

> المطابقة بين المصدرين عبر **اسم اللاعب المُطبّع** + نادي الوجهة (لرفع الدقة). راجع `lib/mergeSources.mjs`.

## شكل «بطاقة REO الموحّدة»

كل حقل = `{ value, _src }` حيث `_src ∈ { 'fotmob', 'transfermarkt', 'missing' }`.

```jsonc
{
  "id": "906937-8634-2026-05-29",
  "name":        { "value": "Anthony Gordon", "_src": "fotmob" },
  "playerImage": { "value": "https://images.fotmob.com/.../906937.png", "_src": "fotmob" },
  "fromClub":    { "value": "Newcastle United", "_src": "fotmob" },
  "fromLogo":    { "value": "https://images.fotmob.com/.../10261.png", "_src": "fotmob" },
  "toClub":      { "value": "Barcelona", "_src": "fotmob" },
  "toLogo":      { "value": "https://images.fotmob.com/.../8634.png", "_src": "fotmob" },
  "fee":         { "value": "€80.0m", "_src": "transfermarkt" },   // الرسوم الرسمية من TM
  "feeFotmob":   { "value": "€80M",   "_src": "fotmob" },          // مرجع FotMob
  "marketValue": { "value": "€75.0m", "_src": "transfermarkt" },
  "date":        { "value": "2026-05-29", "_src": "fotmob" },
  "isToday":     { "value": false, "_src": "fotmob" },
  "age":         { "value": 25, "_src": "transfermarkt" },
  "valueTrendPct": { "value": 25.0, "_src": "transfermarkt" },
  "valueHistory":  { "value": [ ... ], "_src": "transfermarkt" },
  "lastFee":     { "value": "€45.0m", "_src": "transfermarkt" },
  "seasonStats": { "value": { "matches": 38, "goals": 12, "assists": 9, "minutes": 3120 }, "_src": "transfermarkt" },
  "leagueLevel": { "value": 1, "_src": "transfermarkt" },
  "_enriched": true,
  "_matchQuality": "exact",
  "_sources": ["fotmob", "transfermarkt"]
}
```

عند غياب مرجع TM: `_enriched=false`، الرسوم والقيمة من FotMob، وحقول العمق `missing`.

## واجهة الجسر الموحّد

```
GET /api/feed?view=fee|latest        → بطاقات مدموجة من الكاش
GET /api/refresh?view=fee|latest     → التقاط حيّ من FotMob ثم دمج
GET /overview                        → الصورة العامة لكل القوالب
GET /health
```

## نقاط القوة
- **دقة مزدوجة:** رسوم/قيمة من TM + بصريات/حدث من FotMob.
- **تدرّج آمن:** لا تنهار البطاقة عند غياب أحد المصدرين.
- **شفافية كاملة:** كل قيمة تكشف مصدرها (مهم للبث الموثوق).
- **قابلية التوسّع:** أي قالب يستهلك نفس العقد الموحّد.
