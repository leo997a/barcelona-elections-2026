# مراجعة أداة قوالب البث

## أهم المشاكل

1. الترميز العربي غير موحد في بعض الملفات مثل `pages/Editor.tsx` و`pages/Library.tsx` و`index.html`.
   الحل: تنفيذ تمريرة تعريب/Encoding منظمة لاحقا بدل تعديل عشوائي، حتى لا تتغير قيم القوالب أو مفاتيح الحقول بالخطأ.

2. المؤثرات الصوتية كانت محدودة وبعضها يعتمد على روابط خارجية.
   الحل المطبق: إضافة مؤثرات Synth داخلية متعددة مثل `SCOREBUG_SNAP` و`STADIUM_WHOOSH` و`VAR_BUZZ` و`DATA_TICK`، مع اختيار دخول وخروج لكل قالب.

3. انتقالات الظهور والإخفاء كانت مرتبطة بنوع القالب فقط ولا يمكن تخصيصها من المحرر.
   الحل المطبق: إضافة حقول `transitionIn` و`transitionOut` لكل القوالب، مع انتقالات Broadcast متنوعة مثل Scorebug Snap وStadium Sweep وLower Third Wipe.

4. مسار الذكاء الاصطناعي كان يعتمد على موديل تجريبي واحد ويتوقع JSON مثاليا.
   الحل المطبق: جعل `/api/ai` يستخدم `GEMINI_MODEL` عند توفره، ثم يجرب بدائل، ويطلب JSON صريحا، وينظف المخرجات ويتحقق منها قبل إرسالها للواجهة.

5. واجهة التحرير كانت تستورد وظائف AI غير مربوطة فعليا لبعض القوالب.
   الحل المطبق: إضافة زر توليد بيانات مباراة لقوالب Scoreboard، وزر تحويل النص إلى شرائح احترافية لقالب Smart News.

6. الكتالوج كان يحتاج قوالب كروية مباشرة أكثر احترافية.
   الحل المطبق: إضافة حزمة `FOOTBALL_WORLD_FEED` بقوالب أصلية مستوحاة من جودة البث العالمي بدون نسخ هوية قناة محددة.

## القوالب الجديدة

- `كروي عالمي - Scorebug علوي`
- `كروي عالمي - Match Bar فاخر`
- `كروي عالمي - تنبيه VAR`
- `كروي عالمي - تعريف محلل`
- `كروي عالمي - Match Ticker`

## ملاحظات متابعة

- أفضل خطوة لاحقة هي تنظيف الترميز العربي في الواجهة كلها بتمريره واحدة مع اختبار بصري.
- تحذير حجم الحزمة في Vite موجود قبل وبعد التعديل، ويمكن تخفيفه لاحقا بتقسيم الكود.
- يفضل استبدال صور placeholder بروابط أو أصول محلية خاصة بالقناة قبل الاستخدام الإنتاجي.
