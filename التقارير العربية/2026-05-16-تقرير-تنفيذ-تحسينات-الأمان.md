## 1. Rate Limiting لـ `/api/license`

### الملف المعدل: `api/license.ts`

تمت إضافة نظام rate limiting بسيط يعتمد على الذاكرة (in-memory) لكل instance على Vercel:

| الإجراء | الحد | النافذة الزمنية | رمز الحظر |
|---------|------|-----------------|-----------|
| `verify` | 10 محاولات | دقيقة واحدة | 429 |
| `generate` | 3 محاولات | 5 دقائق | 429 |

### كيف يعمل:
- يستخدم `x-forwarded-for` header (يضعه Vercel تلقائياً) لتحديد IP العميل
- `Map<string, {count, resetAt}>` يتتبع المحاولات
- تنظيف تلقائي كل 5 دقائق لمنع تسريب الذاكرة
- الرسائل بالعربية: "محاولات كثيرة. حاول بعد دقيقة." / "محاولات توليد كثيرة. حاول بعد 5 دقائق."

### ملاحظة تقنية:
Rate limiting بالذاكرة على serverless محدود لأن كل instance له ذاكرة منفصلة. لكنه يوفر حماية جيدة ضد brute-force من نفس الجلسة. للحماية الكاملة مستقبلاً يمكن استخدام Vercel KV أو Upstash Redis.

## 2. Security Headers

### الملف المعدل: `api/_lib/http.ts`

أُضيفت 3 headers أمنية إلى كل استجابة API:

| Header | القيمة | الوظيفة |
|--------|--------|---------|
| `X-Content-Type-Options` | `nosniff` | يمنع المتصفح من تخمين نوع المحتوى |
| `Referrer-Policy` | `no-referrer` | يمنع إرسال URL المصدر مع الطلبات |
| `X-Frame-Options` | `DENY` | يمنع تضمين الصفحة في iframe |

تمت الإضافة في:
- `sendJson()` — كل الاستجابات العادية
- `sendMethodNotAllowed()` — استجابات رفض الطريقة

`Cache-Control: no-store` كان موجوداً مسبقاً ✅

## 3. تأمين Cookies للمستقبل

### الملف المعدل: `api/_lib/http.ts`

أُضيفت دالة `setSecureCookie()` جاهزة للاستخدام المستقبلي:

```typescript
setSecureCookie(response, 'session', tokenValue, 86400);
```

| الإعداد | القيمة | الوظيفة |
|---------|--------|---------|
| `HttpOnly` | ✅ | يمنع JavaScript من قراءة الـ cookie |
| `Secure` | ✅ | ترسل فقط عبر HTTPS |
| `SameSite` | `Strict` | تمنع إرسال الـ cookie مع طلبات خارجية (CSRF) |
| `Max-Age` | قابل للتعديل | مدة الصلاحية بالثواني |

**ملاحظة:** الدالة جاهزة ولم تُستخدم بعد. ستُستخدم عند الحاجة لنقل JWT من localStorage إلى cookie.

## 4. تغيير النصوص التعليمية

### الملف المعدل: `pages/Settings.tsx`

**قبل:**
```
GEMINI_API_KEY=your_gemini_key
EDITOR_ADMIN_PASSCODE=choose_a_private_passcode
ADMIN_SESSION_SECRET=choose_a_long_random_secret
```

**بعد:**
```
GEMINI_API_KEY=SET_IN_VERCEL_ONLY
EDITOR_ADMIN_PASSCODE=SET_IN_VERCEL_ONLY
ADMIN_SESSION_SECRET=SET_IN_VERCEL_ONLY
LICENSE_SECRET=SET_IN_VERCEL_ONLY
LICENSE_ADMIN_SECRET=SET_IN_VERCEL_ONLY
```

### الفوائد:
1. لا توجد قيم وهمية يمكن أن يظنها أحد حقيقية
2. إضافة `LICENSE_SECRET` و `LICENSE_ADMIN_SECRET` كتذكير إداري
3. نص واضح يقول: "ضعها في Vercel فقط"
