## سبب مشكلة الإخفاء التلقائي

كان هناك طريقان للتحكم:

- Firebase
- Live API

وكانت إضافة Stream Deck تحاول Firebase أولًا ثم تسقط إلى Live API. هذا يسبب لخبطة عندما يصل أمر من Firebase وأمر آخر من API، خصوصًا مع أمر `toggle_visible`. إذا ضغط الزر أو تكرر الأمر، يظهر القالب ثم يختفي لأن `toggle` يعكس الحالة الحالية بدل أن يضبطها صراحة.

الحل الحالي:

- Stream Deck يرسل رسميًا إلى `/api/live`.
- نافذة العرض تستقبل عبر SSE من `/api/stream` مع polling خفيف احتياطي.
- Firebase لم يعد يتدخل في نافذة العرض.
- الأوامر الأساسية أصبحت `set_visible true` و`set_visible false` بدل الاعتماد على toggle.

## نظام كاش الصور الجديد

الفكرة الصحيحة ليست البحث باسم الصورة مباشرة، بل استخدام ملف هوية:

```json
{
  "id": "cole-palmer",
  "displayName": "Cole Palmer",
  "arabicNames": ["كول بالمر", "بالمر"],
  "club": "chelsea",
  "clubArabic": "تشيلسي",
  "position": "AM/RW",
  "smallImage": "...",
  "renderImage": "...",
  "clubLogo": "..."
}
```

عند كتابة اسم عربي داخل الصندوق الذكي، النظام يطبع احتمالات، يطابق النادي، يعطي ثقة، ثم يختار الصورة المناسبة:

- القوالب السريعة تستخدم `smallImage`.
- قوالب الميركاتو والبطاقات الفخمة تستخدم `renderImage`.
- شعار النادي يأتي من `clubs.json`.
