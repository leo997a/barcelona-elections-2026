## 1. الفكرة العامة وما تم إنجازه

أُضيف إلى الأداة قالبان جديدان كاملان مع 18 قالبًا جاهزًا في المكتبة، ومحرك صوت احترافي
موسّع بـ 30 cue إضافي، وخدمة "الصوت العميق" الاحترافية للنطق الصوتي لقالب
الخبر العاجل. النتيجة: مجموعة بث متكاملة بهوية بصرية وصوتية على مستوى قنوات الميركاتو
الكبرى (مثل Fabrizio Romano / Sky / DAZN).

### القوالب الجديدة

1. **TRANSFER_TARGETS** — أهداف النادي في سوق الانتقالات
   - شريط جانبي يسار قابل للتخصيص (22-50% من العرض، الافتراضي 30%).
   - 6 مراكز كاملة: مهاجم، جناح، وسط، قلب دفاع، ظهير، حارس مرمى.
   - 1-7 لاعبين لكل مركز (اسم + صورة + شعار النادي + اسم النادي + عمر + قيمة + تقييم + جنسية).
   - 3 أوضاع عرض: تتابع (SEQUENCE) / سلايد لمركز واحد / دوران تلقائي بين المراكز.
   - 5 ثيمات بصرية احترافية لكل قالب.
   - 9 قوالب جاهزة في المكتبة (ثلاثة أوضاع × ثلاث ثيمات بارزة لكل وضع).

2. **BREAKING_HERE_WE_GO** — الخبر العاجل / الرسمي / المهم
   - يبدأ بمقدمة درامية (شاشة سوداء + نص ضخم "HERE WE GO" + flash + موجات نبض) ثم
     ينتقل إلى بطاقة الخبر (عنوان، تفاصيل، صورة، مصدر، طابع زمني).
   - 3 متغيرات: BREAKING (أحمر) / OFFICIAL (أزرق) / IMPORTANT (برتقالي).
   - 5 ثيمات بصرية لكل قالب.
   - 9 قوالب جاهزة في المكتبة (3 متغيرات × 3 ثيمات لكل متغير).
   - نظام نطق صوتي احترافي (صوت "Here We Go" فخم وعميق).

### الباقة الصوتية الجديدة (30 صوت)

5 فئات جديدة في صفحة الأصوات (Sounds Page) داخل لوحة التحكم بالبث:
- ⚽ كروية (Football): 8 أصوات
- 🚨 أخبار عاجلة (Breaking): 7 أصوات
- ⚡ ميركاتو (Mercato): 6 أصوات
- 🎞️ سينمائية (Cinematic): 7 أصوات
- ⏱️ أدوات (Utility): 2 أصوات

كل صوت من هذه الـ 30 يمر عبر **Broadcast Pro Synth** الجديد بهوية صوتية موحدة فخمة وعميقة.

## 3. تفاصيل قالب TRANSFER_TARGETS

### 3.1 أوضاع العرض الثلاثة

#### الوضع الأول — SEQUENCE (ظهور متتابع بصوت)
```
[المركز: المهاجمين]
  → بعد 200ms: ظهور اللاعب 1 + صوت TARGET_REVEAL
  → بعد 4 ثوانٍ: ظهور اللاعب 2 + صوت TARGET_REVEAL
  → بعد 4 ثوانٍ: ظهور اللاعب 3 + صوت TARGET_REVEAL
  → ... حتى آخر لاعب (حد أقصى 7)
```
- الفاصل قابل للتعديل من 2 إلى 15 ثانية.
- الصوت لكل هدف قابل للاختيار من قائمة كاملة بالأصوات.
- مثالي للكشف الدرامي عن قائمة الأهداف على الهواء.

#### الوضع الثاني — SLIDE (سلايد لمركز واحد)
```
[المركز المختار من الإعدادات]
  → كل اللاعبين يظهرون دفعة واحدة بانتقال متدرج (stagger 80ms)
```
- اختيار المركز النشط من قائمة منسدلة.
- يصلح للعرض المستمر لمركز محدد طوال المداخلة.

#### الوضع الثالث — CYCLE (دوران تلقائي بين المراكز)
```
[المهاجمين 30 ثانية] → صوت POSITION_SWITCH
[قلب الدفاع 30 ثانية] → صوت POSITION_SWITCH
[الوسط 30 ثانية] → صوت POSITION_SWITCH
... ثم يعود للمهاجمين (loop)
```
- الفاصل قابل للتعديل من 10 إلى 120 ثانية.
- داخل كل مركز يتم كشف اللاعبين بنفس آلية SEQUENCE.
- مؤشرات نقاط في الفوتر تُظهر المركز الحالي.

### 4.4 نظام النطق الصوتي العميق

**ثلاثة أنماط صوت قابلة للاختيار:**

#### أ. BROWSER_TTS_DEEP (موصى به)
- يستخدم Web Speech API بذكاء.
- يبحث في كل أصوات المتصفح عن أعمق صوت متاح.
- معايير اختيار الصوت (نظام نقاط):
  - مطابقة اللغة المطلوبة: -100 نقطة (الأفضل)
  - أسماء ذكورية معروفة (Daniel, Alex, David, Matthew, Brian, Tom, Robert): -40
  - أسماء عربية فخمة (Majed, Tarik, Hamed, Ali, Saleh, Fahad, Omar, Hassan): -40
  - أسماء إسبانية (Carlos, Diego, Jorge, Manuel): -30
  - مفاتيح `male`, `man`, `deep`, `low`, `bass`: -60
  - أصوات معززة (`enhanced`, `premium`, `natural`, `neural`, `wavenet`): -25
  - أصوات نسائية: +80 (مرفوضة)
  - أصوات كوميدية (`novelty`, `whisper`, `cellos`, `zarvox`): +200 (مرفوضة)
- يخفض النغمة (pitch) إلى 0.45 (افتراضياً 1.0) لإحساس أعمق.
- معدل النطق (rate) = 0.82 لإضافة وقار.
- ينتظر voiceschanged event حتى تتحمّل الأصوات.

#### ب. STUDIO_STINGER
- بدون كلام، فقط مؤثر صوتي فخم.
- 4 طبقات صوتية مدمجة:
  1. **Sub bass body** (sine 38→24 Hz) — الوزن العميق
  2. **Body kick** (sawtooth 90→56 Hz + lowpass) — الضربة
  3. **Broadcast riser** (triangle 320→1240 Hz) — الحضور المتوسط
  4. **Air noise sweep** (bandpass 620→2400 Hz) — الـ WHOOSH
- مع plate reverb (32% wet) + tape saturation.

#### ج. FORMANT_PHRASE
- صوت سينمائي صناعي بالكامل (لا يعتمد على المتصفح).
- نظام Two-Formant Vocoder:
  - Glottal source: sawtooth oscillator بنغمة منخفضة (~95 Hz)
  - Formant filters: bandpass على F1 (350-550 Hz) + F2 (900-1500 Hz)
  - Q values عالية (8-9) لتقليد البيئة الصوتية للحلق
- جدول مقاطع جاهز:
  ```
  here   → 0.32s, F1=480, F2=2080
  we     → 0.18s, F1=320, F2=800
  go     → 0.45s, F1=520, F2=920
  breaking → 0.55s, F1=460, F2=1500
  news     → 0.40s, F1=380, F2=1200
  ...
  ```
- لكل مقطع:
  - Glottal source بساوتوث ينخفض من f0 إلى 0.92×f0 (طبيعي للصوت البشري).
  - Envelope: attack سريع 40ms ثم تنازل ناعم.
- لكل العبارة:
  - Low shelf +6dB عند 220 Hz للدفء.
  - Peaking +4dB عند 1700 Hz لوضوح الكلام.
  - Plate reverb send 22% wet.

## 5. محرك Broadcast Pro Synth (الأصوات الجديدة)

### 5.1 سلسلة المعالجة الصوتية (Signal Chain)

```
Source (oscillators + noise)
  ↓
Per-cue bus (ADSR envelope)
  ↓
Tape Saturation (WaveShaper, oversample 2x, k=0.35)
  ↓
Low Shelf (+4dB @ 220Hz)
  ↓
Presence Boost (+3.5dB @ 2.4kHz, Q=1.1)
  ↓
Master Gain → Compressor → Limiter → Destination
  
وبالتوازي عبر send:
  ↓
Plate Reverb (Convolution، per-cue strength 0.06-0.32)
  ↓
Master Gain
```

### 5.2 الأشكال الصوتية الـ 16 (Shapes)

كل صوت في باقة الـ Broadcast Pro يستخدم واحدة من 16 شكلاً صوتياً:

| Shape | الوصف | الطبقات |
|-------|--------|----------|
| **HIT** | ضربة دراماتيكية | sub 38Hz + body 56Hz saw + mid 132Hz + 360Hz + noise sweep 1.8k→220Hz |
| **RISER** | تصاعد متدرج | sub 44Hz + 92→220Hz saw + 240→880Hz triangle + 1080→1480Hz sine + sweep 240→5200Hz |
| **PULSE** | نبض إيقاعي | 3 ضربات square 220Hz/880Hz كل 160ms + sub 50Hz + sweep |
| **SWEEP** | كنس صوتي | sub 60Hz + 880→240Hz triangle + 2 noise sweeps |
| **PING** | رنين دائري | 880Hz + 1320Hz + 1760Hz + sub 110Hz |
| **TICK** | نقرة سريعة | 1320Hz square 45ms + 880Hz square 50ms |
| **REVEAL** | كشف صوتي | sub 38 + 56 saw + 220 triangle + 660 sine + 1320 sine + sweep 1.8k→5.8k |
| **LOCK** | إقفال هدف | 320Hz + 480Hz + 720→1080Hz triangle + sub 56Hz |
| **SCAN** | مسح متدرج | 4 نغمات 620→1040Hz + 1320Hz triangle + bandpass sweep |
| **SWITCH** | تبديل مركز | 220→480 saw + 540→880 triangle + 880 sine + sub 80 |
| **FANFARE** | بوق احتفالي | sub 55 + 220/330/440 square + 660 saw |
| **CHANT** | هتاف جماهيري | sub 55 + 3 ضربات saw 180/220/260 + triangle 360/440/520 + sweep |
| **WHISTLE_S/L** | صفارة قصيرة/طويلة | 2400Hz + 4800Hz sine (افتراضي 0.18s أو 0.7s) |
| **HORN** | بوق ملعب | sub 55 + 110/220 saw + 165/330 saw |
| **BELLS** | أجراس | 660/880/1320Hz + هارمونيات × 2 |
| **DROP** | سقوط دراماتيكي | sub 220 → 36 + sweep عكسي 5.2k→200Hz |
| **GLITCH** | تشويش رقمي | 5 نغمات square عشوائية + bandpass sweep ضيق |

## 9. كيف تستخدم القالبين الآن

### 9.1 إضافة قالب أهداف الانتقالات
1. افتح صفحة Library في الأداة.
2. فلتر "أهداف الانتقالات" أو ابحث عن "Transfer Targets".
3. اختر أحد القوالب الـ 9 الجاهزة (Sequence/Slide/Cycle × ثيم).
4. عدل الإعدادات:
   - **اسم النادي + شعار النادي** (لتغيير الهوية).
   - **عرض الشريط الجانبي** (22-50%).
   - **المراكز المفعلة** (مثلاً `pos1,pos3,pos5` للمهاجم والوسط والظهير فقط).
   - **أهداف كل مركز** (JSON أو سطور).
   - **الفاصل الزمني** للتتابع أو الدوران.
   - **مؤثر صوت الهدف** (TARGET_REVEAL أو أي صوت آخر من القائمة).
   - **إظهار التقييم/الجنسية** (toggles).
5. اضغط Live ليبدأ القالب على البث.

### 9.2 إضافة قالب الخبر العاجل
1. افتح Library.
2. فلتر "Here We Go" أو ابحث عن "Breaking".
3. اختر أحد القوالب الـ 9 (BREAKING/OFFICIAL/IMPORTANT × ثيم).
4. عدل:
   - **النص الكبير** للمقدمة (`headline`، الافتراضي "HERE WE GO").
   - **عنوان وتفاصيل الخبر**.
   - **صورة الخبر** (URL).
   - **المصدر**.
   - **مدة المقدمة** (1000-4500ms، الافتراضي 2200ms).
   - **نمط الصوت**: BROWSER_TTS_DEEP / STUDIO_STINGER / FORMANT_PHRASE.
   - **النص المنطوق + اللغة + العمق + الحجم**.
5. اضغط Live ليبدأ التسلسل الكامل (مقدمة → بطاقة).
