# تقرير عربي - Sportmonks وقوالب الميركاتو وكاش صور اللاعبين

التاريخ: 2026-05-15  
المشروع: `barcelona-elections-2026`  
النطاق: تجربة ربط Sportmonks بطريقة آمنة، إضافة قوالب ميركاتو جديدة، إصلاح صور لاعبي برشلونة، وتقوية مؤثرات الصوت.

## 1. ماذا تغير في هذه الجولة؟

تمت إضافة مسار API جديد داخل الموقع:

```txt
/api/sportmonks/player
```

هذا المسار يعمل كوسيط آمن بين القالب و Sportmonks. القالب لا يضع التوكن في المتصفح، بل يرسل اسم اللاعب أو رقم اللاعب إلى API داخلي، ثم يقوم السيرفر بطلب البيانات من Sportmonks باستخدام:

```txt
SPORTMONKS_API_TOKEN
```

الهدف من هذا التصميم أن تبقى المفاتيح السرية خارج المتصفح وخارج ملفات GitHub.

## 2. كيف نستخدم Sportmonks داخل القوالب؟

في قوالب الميركاتو أضفت حقولًا جديدة:

```txt
sportmonksPlayerId
sportmonksSearch
playerTeam
playerPosition
seasonLabel
playerStatsJson
```

إذا وضعت `sportmonksPlayerId` فالقالب يطلب اللاعب بالرقم.  
إذا تركته فارغًا ووضعت `sportmonksSearch` فالقالب يبحث بالاسم.

المسار الذي يناديه القالب يكون مثل:

```txt
/api/sportmonks/player?search=Lamine%20Yamal
```

أو:

```txt
/api/sportmonks/player?playerId=758
```

عند توفر `image_path` من Sportmonks يمكن للقالب استخدامه كصورة اللاعب، لكن إذا وضعت صورة يدوية في `playerImage` فالصورة اليدوية لها الأولوية.

## 3. هل Sportmonks مجاني؟

Sportmonks ليس مصدرًا مجانيًا بالكامل مناسبًا للإنتاج المستمر. هو API احترافي باشتراكات وتجربة مجانية أو خطة دخول حسب الحساب والعرض الحالي. عمليًا:

- مناسب إذا تريد مصدرًا مرتبًا للاعبين، الفرق، الصور، المواسم، الإحصائيات، الجداول والبطولات.
- ليس بديلًا كاملًا مباشرًا عن WhoScored لكل تفاصيل live match center إلا بعد اختبار تغطية live endpoints المطلوبة.
- الأفضل لقناتك أن نستخدمه كقاعدة بيانات منظمة للاعبين والصور وإحصائيات الموسم، ونستمر في استخدام جسر WhoScored لإحصائيات المباراة الحية إلى أن نثبت مصدر live أفضل.

الحد المهم في API v3 حسب التوثيق الرسمي: الخطط الافتراضية لديها حدود طلبات حسب الكيان، وليس حسب endpoint فقط. لذلك يجب استخدام كاش وعدم طلب نفس اللاعب كل ثانية.

## 4. ماذا أضفت لقوالب الميركاتو؟

تمت إضافة نمطين جديدين داخل `TransferNewsRenderer`:

```txt
PLAYER_SEASON_CARD
PLAYER_IMPACT_CARD
```

وتمت إضافة قالبين جاهزين في المكتبة:

```txt
template-mercato-player-season-card
template-mercato-player-impact-card
```

الأول: بطاقة لاعب موسمية كبيرة تعرض صورة اللاعب، الموسم، المركز، الفريق، 6 إحصائيات أساسية، وقراءة سوق الانتقالات.  
الثاني: بطاقة توقع تأثير اللاعب تعرض الإحصائيات كأشرطة تقدم، مع صورة اللاعب واحتمالية الصفقة.

أضفت أيضًا أيقونات `lucide-react` داخل بطاقات الإحصائيات بدل أن تكون أرقامًا جامدة فقط. الإحصائيات أصبحت تقرأ أسرع للمشاهد: أهداف، تمريرات، تسديدات، تقدم، دقائق، قيمة سوقية، احتمالية.

## 5. إصلاح صور لاعبي برشلونة

سبب المشكلة كان مركبًا:

1. بعض أسماء WhoScored تأتي مختصرة مثل `L. Yamal` أو بدون علامات مثل `Kounde`.
2. الكاش القديم لم يكن يحتوي كل هذه الصيغ.
3. المتصفح أو OBS يمكن أن يحتفظ بنسخة قديمة من ملف الصور.

تم تحديث سكربت الكاش:

```txt
scripts/sync-player-image-cache.mjs
```

وأصبح يولد أسماء بديلة كثيرة لكل لاعب: الاسم الكامل، الاسم بدون علامات، اللقب، أول حرف + اللقب، وهكذا.

تم تحديث ملف:

```txt
public/player-image-cache/barcelona.json
```

ويحتوي الآن على روابط مباشرة من مستودع:

```txt
leo997a/graphicsplayer2026
```

كما تم تغيير رابط الكاش إلى:

```txt
/player-image-cache/barcelona.json?v=20260515
```

حتى لا يقرأ المتصفح النسخة القديمة.

## 6. ماذا عن `REO_ARCHIVE_GITHUB_TOKEN`؟

هذا التوكن ليس خاصًا بـ Sportmonks. وظيفته فقط أن يسمح لجسر Google Cloud بحفظ JSON المباراة في GitHub بعد نهاية المباراة.

المكان الصحيح له:

```txt
Google Cloud VM: /etc/reo-match-bridge.env
```

وليس داخل Vercel، وليس داخل الكود، وليس داخل المتصفح.

المتغيرات المطلوبة للأرشفة:

```txt
REO_ARCHIVE_GITHUB_TOKEN=ضع_التوكن_هنا
REO_ARCHIVE_GITHUB_REPO=leo997a/barcelona-elections-2026
REO_ARCHIVE_GITHUB_BRANCH=main
REO_ARCHIVE_BASE_PATH=match-archive
```

مهم جدًا: GitHub لا يسمح لك برؤية التوكن مرة ثانية بعد إنشائه. إذا ضاع، لا يمكن "جلبه"، بل يجب إنشاء توكن جديد. الصلاحية المطلوبة عادة:

```txt
Repository: leo997a/barcelona-elections-2026
Contents: Read and write
```

بعد التفعيل، عندما تنتهي المباراة يقوم الجسر بحفظ الملف في مسار منظم مثل:

```txt
match-archive/2025-2026/spain-laliga/round-unknown/2026-05-14_1914217_deportivo-alaves-vs-barcelona.json
```

## 7. تطوير الصوت

أضفت مؤثرات صوتية جديدة مولدة داخل المتصفح:

```txt
MERCATO_HIT
CINEMA_BOOM
DATA_SLAM
```

وقمت برفع قوة الماستر للصوت حتى يكون أوضح في البث. هذه ليست ملفات صوت خارجية، بل Web Audio Synth داخلي، وهذا يجعلها سريعة ولا تحتاج تحميل ملفات. لاحقًا يمكننا إضافة مكتبة أصوات WAV احترافية إذا أردت هوية صوتية ثابتة للقناة.

## 8. خطة الكاش الذكية المقترحة للصور والإحصائيات

أفضل نظام عملي:

1. نستخدم GitHub كمستودع صور ثابت:
   `graphicsplayer2026`
2. ننشئ ملف JSON لكل فريق:
   `public/player-image-cache/barcelona.json`
3. القالب يقرأ ملف JSON مرة واحدة فقط عند التحميل.
4. إذا لم يجد اللاعب في GitHub cache، يمكنه تجربة Sportmonks `image_path`.
5. أي صورة تنجح من Sportmonks يمكن لاحقًا حفظها في كاش منظم حتى لا ندفع طلبات متكررة.

بهذا نحصل على:

- سرعة في OBS.
- عدم تقطيع أثناء العرض.
- تقليل طلبات API.
- تحكم كامل بالصور وجودتها.

## 9. أفكار قوالب ميركاتو عصرية للجولات القادمة

1. `Deal Timeline`: خط زمني متحرك من "اهتمام" إلى "عرض رسمي" إلى "فحص طبي" إلى "تمت".
2. `Agent Room`: بطاقة مكالمة وكيل اللاعب مع نسبة الثقة، آخر اجتماع، والموعد المتوقع.
3. `Club War Room`: مقارنة بين 3 أهداف لنفس المركز، كل لاعب له صورة، قيمة، عمر، إحصائيات الموسم.
4. `Medical Reveal`: قالب إعلان اجتياز الفحص الطبي مع صورة اللاعب وبيانات العقد.
5. `Here We Go Meter`: مؤشر ثقة كبير يتحول من بارد إلى ساخن مع حركة سينمائية.
6. `Season Radar`: رادار إحصائي للاعب واحد يقارن آخر موسم بالموسم الحالي.
7. `Transfer Map`: انتقال اللاعب بين بلدين أو ناديين مع مسار ضوئي.
8. `Squad Fit`: كيف يدخل اللاعب في تشكيلة الفريق مع ثلاثة أسباب تكتيكية.

## 11. كيف نستفيد لاحقًا؟

هذه البنية لا تخدم مباراة واحدة فقط. يمكن استخدامها لاحقًا من أجل:

- أرشفة موسم كامل.
- بناء قاعدة بيانات للاعبين والفرق.
- تجهيز قوالب لكل فريق.
- إنشاء مكتبة صور ومعلومات للاعبين.
- إنتاج حلقات ميركاتو بمصدر بيانات منظم.
- مقارنة اللاعبين عبر الموسم.
- استخراج بطاقات جاهزة للنشر على YouTube وInstagram وX.

الخطوة الأقوى التالية هي إضافة لوحة داخل الموقع لإدارة "مصادر اللاعبين": GitHub cache، Sportmonks، إدخال يدوي، ثم اختيار الأولوية بينها لكل قالب.
