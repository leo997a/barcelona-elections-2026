## 🆕 إضافة TEMPLATES_DELETE_GLOBAL

### ما تم

أُضيفت capability جديدة `TEMPLATES_DELETE_GLOBAL` لفصل حذف القوالب العامة عن تعديلها:

| الدور | TEMPLATES_EDIT_GLOBAL | TEMPLATES_DELETE_GLOBAL |
|-------|-----------------------|------------------------|
| OWNER | ✅ | ✅ |
| ADMIN_ASSISTANT | ✅ | ❌ |
| SUBSCRIBER | ❌ | ❌ |
| VIEWER | ❌ | ❌ |

**النتيجة**: ADMIN_ASSISTANT يستطيع **تعديل** القوالب العامة لكن **لا يستطيع حذفها**. فقط OWNER يستطيع الحذف.

تم تحديث `workspace.ts` → `canDeleteTemplate()` لاستخدام هذه القدرة الجديدة.

### اختبار 2: `/api/admin/secrets` — حماية الوصول ✅

#### بدون Authorization header:

```
POST /api/admin/secrets {"action":"status"}
→ HTTP 401: {"error":"Authorization required."}  ✅
```

#### بدون Authorization + action test:

```
POST /api/admin/secrets {"action":"test","service":"all"}
→ HTTP 401: {"error":"Authorization required."}  ✅
```

#### مع token مزيف:

```
POST /api/admin/secrets
Authorization: Bearer fake.token.here
→ HTTP 401: {"error":"Invalid or expired session."}  ✅
```

#### ملاحظة مهمة
- **لم يُعرض أي secret كامل** في أي رد ✅
- كل الردود تحتوي فقط metadata (exists, length, last4, fingerprint)
