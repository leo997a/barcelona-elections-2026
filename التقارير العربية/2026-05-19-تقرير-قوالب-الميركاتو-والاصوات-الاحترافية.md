## 5. نظام الصوت الاحترافي — deepVoiceSynth v3

### 5.1 الملف
`services/deepVoiceSynth.ts` — 550 سطر

### 5.2 المشاكل التي حُلّت

**المشكلة 1 — التداخل الصوتي:**
تم حلها بنظام **Mutex كامل**. كل مرة تُشغَّل إعلاناً جديداً:
1. يُلغى الـ AbortController للـ fetch الجاري
2. يُوقف الـ Studio Stinger فوراً
3. تُوقف كل الـ AudioBufferSourceNodes
4. تُبدأ جلسة جديدة نظيفة

**المشكلة 2 — تداخل العربي والإنجليزي:**
تم حلها بـ **Auto Language Segmentation**:
```
"Here we go — هنا نذهب"
  ↓ تقسيم تلقائي
  "Here we go" → Primary Lane (en-US-GuyNeural)
  "هنا نذهب"   → Secondary Lane (ar-SA-HamedNeural)
  ↓ تشغيل متسلسل بدون تداخل
```

**المشكلة 3 — جودة الصوت العربي:**
تم حلها بـ **Microsoft Edge Neural TTS** عبر StreamElements.
الأصوات العربية المتاحة:
- 🇸🇦 **Naayf** — عربي رجالي طبيعي (StreamElements)
- 🇸🇦 **ar-SA-HamedNeural** — فصيح ضخم (Edge Neural)
- 🇪🇬 **ar-EG-ShakirNeural** — مصري احترافي (Edge Neural)
- 🇦🇪 **ar-AE-HamdanNeural** — إماراتي (Edge Neural)
- 🇸🇦 **ar-SA-ZariyahNeural** — سعودي نسائي (Edge Neural)
- 🇪🇬 **ar-EG-SalmaNeural** — مصري نسائي (Edge Neural)

### 5.3 سلسلة المزودين (Fallback Chain)
```
1. Microsoft Edge Neural (StreamElements endpoint)
   ↓ إذا فشل
2. Google Translate TTS (للجمل القصيرة < 200 حرف)
   ↓ إذا فشل
3. Studio Stinger فقط (بدون كلام)
```
**لا يوجد browser speechSynthesis** — كان هو سبب الصوت السيئ.

### 5.4 Broadcast Bus — معالجة الصوت
كل صوت يمر عبر:
```
MP3 Source
  ↓
Low Shelf +5dB @ 220Hz (دفء الصوت)
  ↓
Presence Boost +4dB @ 2.4kHz (وضوح الكلام)
  ↓
Tape Saturation WaveShaper (دفء أنالوغي)
  ↓
Plate Reverb 18% wet (إحساس استوديو)
  ↓
Master Output
```

### 5.5 Studio Stinger — المؤثر الصوتي تحت الصوت
4 طبقات تعمل معاً:
1. **Sub bass** (sine 38→24 Hz) — الوزن العميق
2. **Body kick** (sawtooth 90→56 Hz + lowpass) — الضربة
3. **Broadcast riser** (triangle 320→1240 Hz) — الحضور
4. **Air noise sweep** (bandpass 620→2400 Hz) — الـ WHOOSH

### 5.6 إعدادات الصوت في القالب
| الحقل | الوصف |
|-------|--------|
| `primaryProvider` | مصدر الصوت الأساسي (إنجليزي/لاتيني) |
| `primaryVoiceId` | الصوت المختار (Guy, Davis, Ryan...) |
| `primaryCustomUrl` | رابط MP3 مخصص للصوت الأساسي |
| `secondaryProvider` | مصدر الصوت الثانوي (عربي) |
| `secondaryVoiceId` | الصوت العربي (حامد، شاكر، زاريا...) |
| `secondaryCustomUrl` | رابط MP3 مخصص للعربي |
| `voiceVolume` | حجم الصوت 0-1.4 |

### 7.4 القالب الثالث: MERCATO_BUDGET_TRACKER — متتبع ميزانية النادي

**الفكرة:** دفتر حسابات النادي في الميركاتو — صفقات واردة (مبيعات) وصادرة (شراء) مع رصيد متبقٍ وبار نسبة الميزانية.

**التصميم:**
- **أعلى:** شعار النادي + اسمه + الموسم
- **إحصائيات:** 4 بطاقات (الميزانية الأولية / الوارد / الصادر / المتبقي)
- **بار:** نسبة الميزانية المتبقية مع لون يتغير
- **جدول:** كل صفقة في سطر (نوع IN/OUT + اللاعب + النادي + التاريخ + المبلغ)

**صيغة البيانات:**
```json
[
  { "type": "out", "player": "João Cancelo",  "club": "Man City",      "amount": 25, "date": "Jul 03" },
  { "type": "in",  "player": "João Félix",    "club": "Atletico",      "amount": 12, "date": "Jul 11" }
]
```

**الحسابات التلقائية:**
- `totalIn` = مجموع الصفقات الواردة
- `totalOut` = مجموع الصفقات الصادرة
- `remaining` = initialBudget + totalIn - totalOut
- `remainingPct` = (remaining / initialBudget) × 100

### 7.5 القالب الرابع: MERCATO_DEADLINE_DAY — يوم انتهاء الميركاتو

**الفكرة:** عداد عكسي حي ليوم إغلاق الميركاتو + تغذية صفقات مباشرة مصنفة (Breaking / Done / Rumor).

**التصميم:**
- **شريط hazard tape** أعلى وأسفل (أصفر/أسود)
- **يسار:** عنوان DEADLINE DAY ضخم + العنوان الفرعي + مؤشر LIVE
- **يمين:** عداد عكسي حقيقي HH:MM:SS يتحدث كل ثانية
- **إحصائيات:** 3 بطاقات (Breaking / Completed / Rumors)
- **جدول:** تغذية صفقات مباشرة مع الوقت + اللاعب + النادي المغادر → الجديد

**صيغة البيانات:**
```json
[
  { "time": "14:08", "player": "João Cancelo",    "from": "Man City",  "to": "Barcelona", "status": "done" },
  { "time": "15:32", "player": "Bruno Guimarães", "from": "Newcastle", "to": "Man City",  "status": "breaking" },
  { "time": "16:11", "player": "Frenkie de Jong", "from": "Barcelona", "to": "Liverpool", "status": "rumor" }
]
```

**الإعداد الرئيسي:**
- `targetTime` — وقت الإغلاق بصيغة ISO (مثلاً `2026-08-31T23:00`)
- العداد يحسب الفرق الحقيقي بين الوقت الحالي والهدف

  - خط مسح متحرك على الصورة (scanning effect)
  - أركان bracket للإطار
- **وسط:** رادار SVG كامل:
  - حلقات شبكية (25%, 50%, 75%, 100%)
  - محاور لكل قابلية
  - خط مسح دوار (360° كل 4 ثوانٍ)
  - مضلع ملون بالقابليات
  - نقاط على كل محور
  - تسميات خارج الرادار
- **يمين:**
  - 6 أشرطة قابليات مع تسمية + قيمة + لون
  - Heat map للمناطق الأكثر نشاطاً على الملعب
  - Verdict نهائي بإطار مميز

**صيغة بيانات الرادار:**

```json

```
- `x`, `y` — موقع النقطة كنسبة مئوية من الملعب (0-100)
