# تقرير تنفيذ جسر الميركاتو الأونلاين

التاريخ: 2026-06-30

## الهدف

تحويل قوالب الميركاتو إلى نفس فكرة قوالب المونديال: القالب لا يعتمد على رابط محلي مثل `127.0.0.1`، بل يطلب البيانات من مسار ثابت داخل نفس الموقع:

`/api/mercato`

هذا المسار هو طبقة أمان وتوحيد. إذا كان الجسر الخارجي منشورًا ومضبوطًا عبر متغيرات البيئة يجلب البيانات الحقيقية، وإذا لم يكن مضبوطًا يرجع بيانات احتياطية حتى لا ينكسر العرض.

## ما تم تنفيذه

1. إضافة API داخلي:
   - الملف: `api/mercato.ts`
   - المسار: `/api/mercato`
   - يدعم: `view=fee|latest|probability|medical|sources`
   - يدعم: `limit`, `player`, `club`
   - يرجع تشخيصًا واضحًا: هل الجسر مضبوط، هل تم استدعاء المصدر، وهل البيانات من الجسر أم احتياطية.

2. ربط السيرفر:
   - الملف: `server/server.ts`
   - تمت إضافة `/api/mercato` بجانب `/api/reo-match`.

3. حماية القوالب من الروابط المحلية:
   - الملف: `utils/data/useTemplateData.ts`
   - أي رابط قديم مثل `http://127.0.0.1:4319` أو `/api/feed` يتم تحويله داخل القالب إلى `/api/mercato`.

4. إضافة إعدادات بيانات للميركاتو:
   - `bridgeApiUrl`: الافتراضي `/api/mercato`
   - `dataBridgeView`: نوع البيانات
   - `manualRefreshNonce`: تحديث يدوي
   - `liveRefreshEnabled`: التحديث المباشر، وهو متوقف افتراضيًا
   - `pollIntervalSec`: فترة التحديث عند تشغيل المباشر

5. تجهيز جسر قابل للنشر:
   - المجلد: `cloud/reo-mercato-bridge`
   - مأخوذ من حزمة العمل الخارجي، ومعدل ليعمل على `0.0.0.0` مع `PORT`.
   - يدعم Bearer Token عبر:
     - `REO_TRANSFERS_BRIDGE_TOKEN`
     - أو `REO_BRIDGE_TOKEN`
   - يحتوي تعليمات نشر في:
     - `cloud/reo-mercato-bridge/ONLINE-DEPLOY.md`

## متغيرات البيئة المطلوبة على الموقع

داخل Hostinger أو أي بيئة نشر للتطبيق:

```env
REO_TRANSFERS_BRIDGE_URL=https://YOUR-MERCATO-BRIDGE/api/feed
REO_TRANSFERS_BRIDGE_TOKEN=ضع_نفس_التوكن_هنا
REO_TRANSFERMARKT_BRIDGE_URL=
REO_TRANSFERMARKT_BRIDGE_TOKEN=
```

ملاحظة: إذا لم تضبط هذه القيم سيعمل `/api/mercato` بالبيانات الاحتياطية، وهذا مقصود حتى لا يتعطل القالب أثناء البث.

## نتيجة الفحص

- `npm run build`: ناجح.
- استدعاء مباشر للـ API المبني: ناجح.
- نتيجة الاختبار بدون جسر منشور:
  - `statusCode: 200`
  - `service: reo-mercato-details`
  - `responseMode: fallback`
  - `bridgeConfigured: false`
  - أول بطاقة اختبار: `Nico Williams`

## ما تبقى

1. نشر `cloud/reo-mercato-bridge` على خدمة Node دائمة.
2. وضع رابط الجسر المنشور في متغير `REO_TRANSFERS_BRIDGE_URL`.
3. إعادة نشر التطبيق على Hostinger.
4. اختبار القالب من رابط الموقع الحقيقي والتأكد من أن `responseMode` أصبح `bridge` بدل `fallback`.
