# تقرير إصلاح الصندوق الذكي والصوت والمزامنة

تاريخ التنفيذ: 2026-05-15

## ما المشكلة التي ظهرت؟

- أخطاء `overbridgenet.com/jsv8/offer` و Sentry ليست من ملفات الموقع. الفحص داخل المستودع لم يجد أي استدعاء لهذه العناوين، لذلك هي غالبا من إضافة متصفح أو سكربت خارجي.
- الخطأ الحقيقي من داخل النظام كان في Firebase:
  - تسجيل الدخول المجهول فشل بسبب `auth/admin-restricted-operation`.
  - بعد هذا الفشل كان النظام يحاول الكتابة إلى مسار `public-live-output`.
  - قاعدة Firebase رفضت الكتابة، فظهرت رسائل `PERMISSION_DENIED` بشكل متكرر.

## ماذا أصلحت؟

1. جعلت `syncManager` يفهم أن Firebase غير قابل للكتابة عند فشل المصادقة أو صلاحيات قاعدة البيانات.
2. عند حدوث هذا، النظام يتحول تلقائيا إلى المسار العملي:
   - التخزين المحلي داخل المتصفح.
   - واجهة `/api/live` التي تغذي نافذة العرض.
   - بدون تكرار أخطاء `PERMISSION_DENIED` في كل تحديث.
3. أوقفت تفعيل Firebase الافتراضي المدمج لأن المشروع الحالي يرفض anonymous auth. دعم Firebase اليدوي بقي موجودا إذا تم إدخال إعداد صحيح لاحقا، لكن المسار الافتراضي الآن هو Live API و SSE.
4. أضفت شبكة أمان في `/api/ai`:
   - إذا فشلت كل مفاتيح Gemini أو انتهى الكريدت، الصندوق الذكي لا ينهار.
   - يرجع نتيجة محلية ذكية للأخبار وقوالب الميركاتو.
5. دعمت الجملة:
   - `ليفاندوفسكي قريب من مغادرة برشلونة بنسبة 65% مجانا بعد نهاية عقده`
   - يتم فهمها كالتالي:
     - اللاعب: Robert Lewandowski
     - النادي المصدر: Barcelona
     - الوجهة: Free agent
     - قيمة الصفقة: Free transfer / end of contract
     - نسبة الاحتمال: 65%
6. حسنت ربط صور اللاعبين:
   - الأولوية لكاش GitHub الخفيف والسريع.
   - إذا لم توجد صورة لليفاندوفسكي في الكاش، يوجد fallback من SportRenders.
   - تم التأكد أن صورة GitHub وصورة SportRenders ترجعان `200 OK`.
7. أضفت مؤثرات صوت ميركاتو أقوى:
   - `HERE_WE_GO_STING`
   - `CONTRACT_STAMP`
   - `AGENT_CALL`
   - `CASH_REGISTER`
   - `MEDICAL_PASS`
   - `RUMOUR_GLITCH`
   - `DEAL_LOCK`
   - `ULTRA_RISER`
8. أضفت قالبين جديدين للميركاتو:
   - `Mercato - Contract Exit Watch`
   - `Mercato - Here We Go Board`

## لماذا هذا أفضل؟

- الصندوق الذكي لم يعد معتمدا على نجاح Firebase حتى يعمل داخل المحرر.
- لو فشل الذكاء السحابي مؤقتا، يبقى هناك fallback عملي بدل ظهور خطأ للمستخدم.
- صور اللاعبين أصبحت تمر عبر ترتيب واضح: كاش سريع أولا، ثم fallback خارجي عند الحاجة.
- الصوت أصبح فيه خيارات أقوى ومناسبة لفقرات الانتقالات.

## ملاحظات تطوير لاحقة

- الأفضل لاحقا بناء خدمة كاش صور كاملة تحفظ صور SportRenders أو أي مصدر خارجي في مستودع الصور أو تخزين خاص، بدل الاعتماد الطويل على hotlink مباشر.
- يمكن توسيع قاموس اللاعبين والأندية تدريجيا من كاش `graphicsplayer2026`.
- يمكن إضافة نمط عرض جديد خاص فقط بـ `Contract Exit Watch` بدل استخدام `DEAL_BREAKER` الحالي، ليصبح عندنا شكل بصري مخصص لعقود نهاية الموسم.
