# تقرير عربي - تحسينات إحصائيات المباراة

التاريخ: 2026-05-14

## ما تم تنفيذه

- تم إصلاح حساب "التمريرات المفتاحية" في القالب والجسر السحابي بحيث لا يعتمد فقط على نوع حدث `KeyPass`، بل يقرأ إشارات `keyPass` و`assist` و`chance created` وبيانات `satisfiedEventsTypes` و`qualifiers` عندما تصل من WhoScored.
- تم جعل إحصائيات اللاعبين أكثر ذكاء: لا تظهر مجموعات كلها صفر، ولا تظهر قوائم صناع/مدافعين إذا لا توجد قيمة فعلية.
- تم تحويل عرض إحصائيات المباراة إلى صفحات صغيرة من 3 إحصائيات داخل كل دورة حتى لا يحدث قص أو تداخل في الفريم.
- تم الحفاظ على أكثر من 20 إحصائية في الدورة الكاملة عبر تقسيم المجموعات إلى صفحات متتابعة كل 30 ثانية.
- تم إضافة أزرار سريعة داخل محرر القالب لاختيار نوع إحصائيات المباراة: ذكي، هجوم، تمرير، دفاع، حراسة، الكل.
- تم إضافة أزرار سريعة لاختيار نوع إحصائيات اللاعبين: ذكي، تسديد، تمرير، دفاع، حراس، الكل.
- تم إضافة اختيار اتجاه إحصائيات الفريقين: المضيف يسار أو الضيف يسار، مع زر لعكس ألوان الفريقين.
- تم إضافة دعم صور اللاعبين بطريقة آمنة عبر حقل `playerImageMapJson`، مع تحميل كسول وفشل ناعم إذا الصورة غير متاحة.
- تم إيقاف انتقالات اللوحات افتراضيا عبر `enablePanelTransitions=false` لتقليل التقطيع أثناء البث.
- تم تحديث نسخة أداة الاستخراج المحلية في `C:\New folder\reo_show_broadcast_identity\match-stats\extractor\extract_match.py` بنفس منطق الجسر السحابي.

## طريقة مفتاح التحكم

مفتاح فتح التحكم لا يؤخذ من داخل المتصفح. يجب وضعه في Vercel كمتغير بيئة باسم:

- `EDITOR_ADMIN_PASSCODE`
- أو `ADMIN_ACCESS_CODE`

إذا نسيته، لا نحتاج معرفة القديم. الأفضل تعيين قيمة جديدة في Vercel ثم إعادة نشر الموقع.

## ملاحظات الصور

تم تجهيز القالب ليستقبل روابط صور اللاعبين من JSON مثل:

```json
{
  "Pedri": "https://example.com/pedri.png",
  "12345": "https://example.com/player-12345.png"
}
```

الأفضل هندسيا عدم الاعتماد على تحميل الصور مباشرة من موقع خارجي أثناء البث. المرحلة التالية المناسبة هي بناء كاش صور صغير خاص بنا، ثم وضع روابط الكاش في `playerImageMapJson`.

## التحقق

- تم تشغيل `npm run lint` بنجاح.
- تم تشغيل `npm run build` بنجاح.
- تم تشغيل `python -m py_compile` لملفات الجسر السحابي بنجاح.
- تم تشغيل `python -m py_compile` لنسخة أداة الاستخراج المحلية بنجاح.
- تم فحص القالب محليا عبر المتصفح على `http://127.0.0.1:5173` وتأكدت أن اللوحات لا تتداخل في وضع العرض التجريبي.
