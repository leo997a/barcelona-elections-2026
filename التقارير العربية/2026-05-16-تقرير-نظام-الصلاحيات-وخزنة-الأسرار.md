### الأسرار المراقبة

### Google Secret Manager

- **الحالة**: Skeleton فقط — لم يُنفذ بعد.
- **التفعيل**: `SECRET_PROVIDER=google_secret_manager`
- **الوضع الافتراضي**: `SECRET_PROVIDER=env` (أو بدون المتغير)
- **لا يوجد**: credentials أو service account JSON في الكود
- **Build لا يفشل** إذا Google غير مفعّل

### كيفية الانتقال لاحقاً

1. أنشئ مشروع Google Cloud مع Secret Manager API
2. أنشئ Service Account بصلاحية `secretmanager.versions.access`
3. نفّذ `GoogleSecretManagerProvider` فعلياً
4. انقل سر واحد في كل مرة واختبر
5. لا تغيّر tokens VPS تلقائياً — يدوياً فقط

**الحماية**: JWT admin session مطلوب.

**الاختبارات المتاحة**:

    "latencyMs": 142

**ممنوع**: لا يعرض أي token أو قيمة سرية.

## استراتيجية عدم ضياع المفاتيح

### الحالة الحالية
- مفتاح واحد (REO_PLAYER_STATS_BRIDGE_TOKEN) يربط Vercel بـ VPS
- إذا ضاع — يتوقف الربط

### الحل المنفذ: Rotation Checklist

1. **ولّد** توكن جديد (Candidate) من لوحة التحكم
2. **انسخه فوراً** — يظهر مرة واحدة فقط
3. **ضعه في Vercel** → REO_PLAYER_STATS_BRIDGE_TOKEN
4. **ضعه على VPS** → REO_PLAYER_STATS_TOKEN
5. **أعد النشر** — Vercel + VPS
6. **اختبر** — اضغط "اختبار الاتصال" وتأكد من `auth.valid=true`
7. **أكد** التدوير

### ما لم يُنفذ (مقصوداً)
- لا يوجد زر "Show Current Token"
- لا يتغير token تلقائياً
- لا يوجد multi-token على VPS الآن
- لا يتصل بـ Vercel API لتغيير env vars تلقائياً
