## المحور الأول: تحسينات الصوت المتقدمة

### 1. Safety Limiter (جدار أمان الصوت)

**المشكلة:** الصوت يمكن أن يتجاوز الحد ويسبب تشويش (clipping) خصوصاً عند رفع Master Volume.

**الحل:** أضفنا `DynamicsCompressorNode` كـ Limiter بعد الـ Compressor:

| المعامل | القيمة | الوظيفة |
|---------|--------|---------|
| threshold | -1 dB | الحد الأقصى قبل القطع |
| knee | 0 | استجابة فورية (brick-wall) |
| ratio | 20:1 | ضغط شبه كامل |
| attack | 0.001s | استجابة فورية |
| release | 0.05s | تحرير سريع |

**السلسلة الصوتية الجديدة:**
```
masterGain → compressor → limiter → destination
```

### 2. تخصيص Reverb/SubBass لكل Cue

**المشكلة:** كل الأصوات كانت تأخذ نفس نسبة الـ reverb (18%) ونفس subBass، مما يجعل أصوات الخروج (مثل SOFT_FADE) ثقيلة بلا داعي.

**الحل:** خريطة `CUE_FX_CONFIG` تحدد لكل صوت نسبته:

| الصوت | Reverb | SubBass |
|-------|--------|---------|
| CINEMA_BOOM | 32% | 1.2x |
| GOAL_HORN | 28% | 1.0x |
| HERE_WE_GO_STING | 26% | 1.1x |
| MERCATO_HIT | 24% | 1.0x |
| SCOREBUG_SNAP | 10% | 0.42x |
| DATA_TICK | 8% | 0.30x |
| SOFT_FADE | 4% | 0.0x (بدون!) |
| BROADCAST_OUT | 6% | 0.0x |

**النتيجة:** الأصوات الكبيرة تأخذ عمق وقوة، والأصوات الخفيفة تبقى نظيفة.

### 3. لوحة Preview Sound في الإعدادات

**ما تم إضافته:**

- **شريط Master Volume**: من 0% إلى 150% مع تغيير فوري
- **شبكة معاينة الأصوات**: 25 مؤثر مصنفة في 3 فئات:
  - 🎙️ أصوات البث (10 أصوات)
  - ⚡ أصوات الميركاتو (8 أصوات)
  - 🎬 أصوات خاصة (7 أصوات)
- كل زر يشغّل الصوت فوراً مع تأثير بصري

## المحور الثاني: إصلاح نظام الحماية

### مشكلة المفاتيح المفقودة (كانت كارثة!)

**المشكلة:** عند توليد مفتاح جديد من لوحة التحكم، المفتاح يُعرض فقط ولا يُحفظ. المستخدم يفقده عند تحديث الصفحة.

**السبب:** دالة `handleGenerateKey` كانت تعمل `setGeneratedKey(data.key)` فقط بدون استدعاء `licenseService.activate()`.

**الإصلاح:**
1. بعد التوليد، يتم استدعاء `licenseService.activate(data.key)` تلقائياً
2. إذا فشل التفعيل عبر API (مثلاً أوفلاين)، يُحفظ مباشرة في `localStorage` كـ fallback
3. أُضيف زر "إعادة تحميل بالمفتاح الجديد" بجانب زر النسخ
4. النص أصبح "✅ المفتاح جاهز ومحفوظ تلقائياً"

## المحور الثالث: تحويل الـ Routing إلى URL Paths

### المشكلة

الموقع كان يستخدم state-based routing داخلي:
```
route === 'settings' → يعرض Settings
```
لكن الـ URL يبقى `/` دائماً. لذلك `barcelona-elections-2026.vercel.app/Settings` يعطي **404**.

### الحل

1. **`vercel.json`**: أضفنا SPA rewrites:
```json
{
  "rewrites": [
    { "source": "/api/(.*)", "destination": "/api/$1" },
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

2. **`App.tsx`**: حوّلنا الـ routing:
   - `pathnameToRoute()` تقرأ URL وتحوله إلى route
   - `setRoute()` تستخدم `window.history.pushState()` لتحديث URL
   - `popstate` event listener للتعامل مع زر Back

3. **المسارات المدعومة الآن:**

| المسار | الصفحة |
|--------|--------|
| `/` | الرئيسية |
| `/Library` | المكتبة |
| `/Operator` | غرفة التحكم |
| `/Integrations` | الربط الخارجي |
| `/Settings` | الحماية والإعدادات |

## المحور الرابع: نقل اللاعبين إلى players.json

### المشكلة

ملف `playerIdentity.ts` تضخم إلى 377 سطر مع 18 لاعب مشفّر يدوياً.

### الحل

1. **`public/identity/players.json`**: ملف JSON يحتوي كل اللاعبين (18 لاعب) مع URLs كاملة
2. **`playerIdentity.ts`**: أصبح يحتوي 4 لاعبين فقط كـ seed
3. **`loadPlayersFromJson()`**: دالة async تحمّل الملف عند أول استخدام وتضيف اللاعبين الجدد
4. **`TransferNewsRenderer`**: يستدعي `loadPlayersFromJson()` عند تحميل قوالب الميركاتو

**الفائدة:**
- إضافة لاعبين جدد = تعديل JSON فقط
- لا حاجة لإعادة بناء التطبيق
- حجم الحزمة أصغر
