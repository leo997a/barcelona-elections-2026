### التنفيذ التقني

**RendererProps.playSound** تم تقييده على مستوى TypeScript:

```typescript
// قبل (يسمح بـ ENTRY/EXIT — خطير):
playSound: (type: 'ENTRY' | 'TRANSITION' | 'EXIT') => Promise<void>;

// بعد (يسمح فقط بـ TRANSITION — آمن):
playSound: (type: 'TRANSITION') => Promise<void>;
```

هذا يعني: **أي مطور يحاول كتابة `playSound('ENTRY')` سيحصل على خطأ TypeScript فوراً**.

## الجزء الثاني: حماية React Strict Mode 🛡️

### المشكلة
React Strict Mode في التطوير يعيد تنفيذ `useEffect` مرتين. بدون حماية:
- ENTRY يُشغّل مرتين
- EXIT يُشغّل مرتين
- أصوات مكررة حتى بعد حذف التكرار من renderers

### الحماية المُطبّقة (3 طبقات)

#### 1. `previousVisibilityRef` — حالة سابقة حقيقية
```typescript
const previousVisibilityRef = useRef<boolean | null>(null);
// يتحقق أن الانتقال فعلي (false→true أو true→false)
// يتجاهل إعادة التنفيذ (true→true) من Strict Mode
```

#### 2. `lastSoundAt` — debounce 300ms
```typescript
const lastSoundAt = useRef(0);
const SOUND_DEBOUNCE_MS = 300;
// يمنع تشغيل نفس الصوت إذا مر أقل من 300ms
// يحمي من أوامر متتالية سريعة (Stream Deck spam)
```

#### 3. `lastCommandId` — حماية unmount
```typescript
const lastCommandId = useRef(0);
const commandId = ++lastCommandId.current;
// يضمن أن setTimeout لا يطبق unmount من أمر قديم
// يحمي من race condition عند TAKE IN → TAKE OUT بسرعة
```

### تسلسل الحماية

```
config.isVisible يتغير
    ↓
هل prevVisible !== true? (Strict Mode guard)
    ↓ نعم
هل مر > 300ms من آخر صوت? (debounce guard)
    ↓ نعم
شغّل ENTRY + حفظ timestamp
    ↓
إذا EXIT: setTimeout مع commandId guard
```

## الجزء الثالث: تأخير TTS 🎤

### المشكلة
ExclusiveAlert كان يشغل TTS بعد 400ms. هذا يتداخل مع صوت ENTRY.

### الحل
```
ENTRY sound (من OverlayRenderer)
    ↓ ~500ms (مدة synth cue)
↓ 200ms فراغ آمن
TTS يبدأ عند 700ms
```

تم تغيير `setTimeout(..., 400)` → `setTimeout(..., 700)`

### كيف يقرأ player-stats-bridge من cache

```javascript
// في reo-player-stats-bridge/server.js
const CACHE_DIR = '/opt/reo-data-cache/fbref';

app.get('/api/player/:name', (req, res) => {
  const files = fs.readdirSync(CACHE_DIR);
  for (const file of files) {
    const data = JSON.parse(fs.readFileSync(path.join(CACHE_DIR, file)));
    const player = data.find(p => 
      p.player.toLowerCase().includes(req.params.name.toLowerCase())
    );
    if (player) return res.json({ status: 'ok', data: player });
  }
  // لا بيانات → لا يخترع أرقام
  res.json({ status: 'unavailable', warning: 'Player not found in cache' });
});
```

### كيف نمنع scraping أثناء البث

| الآلية | الوصف |
|--------|-------|
| **Cron timing** | يعمل 06:00 و 12:00 UTC فقط (بعيد عن أوقات البث) |
| **Manual trigger** | `ssh vps && /opt/reo-datafootball-worker/run.sh` |
| **Lock file** | worker يكتب `/opt/reo-data-cache/.lock` أثناء التحديث |
| **Bridge reads only** | bridge لا يشغل أي scraping — يقرأ JSON فقط |
| **No Selenium** | FBref يعمل بـ requests فقط (لا Chrome headless) |
| **rate limiting** | 5 ثوانٍ بين كل طلب لمنع 403 |
