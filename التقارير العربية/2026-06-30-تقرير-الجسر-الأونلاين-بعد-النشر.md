# تقرير الجسر الأونلاين للميركاتو بعد النشر

التاريخ: 2026-06-30

## النتيجة

تم نشر جسر الميركاتو الأونلاين على Google Cloud Run وربطه كمسار افتراضي داخل API الأداة، بحيث لا يعتمد عرض قوالب الميركاتو على `localhost` أو جهاز التشغيل المحلي.

رابط الجسر:

`https://reo-mercato-bridge-871752181341.us-central1.run.app`

- المشروع: `banded-setting-475000-i8`

## الربط داخل الأداة

تم تعديل `api/mercato.ts` ليستخدم جسر Cloud Run تلقائيًا عندما لا تكون قيمة `REO_TRANSFERS_BRIDGE_URL` موجودة في البيئة.

هذا مهم لأن Hostinger قد لا يحتوي متغيرات البيئة بعد، ومع ذلك ستبقى القوالب قادرة على جلب بيانات الميركاتو من الجسر الأونلاين.

المؤشرات المتوقعة من `/api/mercato` بعد نشر الواجهة:

- `responseMode: bridge`
- `bridgeUrlDefaultUsed: true`
- `upstreamStatus: 200`

## تحقق Hostinger بعد الدفع

بعد دفع commit إلى GitHub، تم فحص مسار Hostinger مباشرة:

`https://peachpuff-herring-712997.hostingersite.com/api/mercato?view=fee&limit=2`

النتيجة:

- `responseMode: bridge`
- `bridgeUrlDefaultUsed: true`
- `upstreamStatus: 200`
- `count: 2`
- أول صفقة في العينة: `Anthony Gordon`

هذا يؤكد أن النسخة الحية لم تعد تستخدم fallback في مسار الميركاتو الأساسي.

## ملاحظات تشغيلية

- فشل مسار النشر المباشر عبر Artifact Registry سابقًا بسبب `Container import failed`.
- تم تجاوز المشكلة باستخدام `gcr.io` مع صورة منشورة يدويًا.
- نسخة الجسر الحالية تحتوي Chromium داخل الحاوية، لذلك يعمل مسار التحديث المباشر `/api/refresh` بدل الاعتماد فقط على snapshot ثابت.

## القادم

1. دفع تغييرات الربط والتقرير إلى GitHub.
2. انتظار Auto Deploy في Hostinger.
3. فحص `https://peachpuff-herring-712997.hostingersite.com/api/mercato?view=fee&limit=2`.
4. العودة لقوالب المونديال والميركاتو قالبًا قالبًا، مع أولوية:
   - ضبط التشكيلة ومواقع اللاعبين.
   - فصل الثيمات عن الستايلات.
   - تحسين الأنيميشن الداخلي والانتقالات.
   - ربط البيانات الحية بدون كلمات مصدر ظاهرة داخل التصميم.
