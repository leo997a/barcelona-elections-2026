# تقرير عربي - ترقية قالب إحصائيات المباراة

## ما تم تطويره

- رفع أحداث المباراة إلى أعلى لوحة إحصائيات المباراة لأنها أهم معلومة أثناء البث.
- زيادة عدد إحصائيات الفريقين في كل صفحة دوران من 3 إلى 5 إحصائيات.
- إضافة أيقونات حديثة لكل مجموعة ولكل سطر إحصائي حتى لا يبدو العرض كجدول جامد.
- توضيح رقم اللاعب المؤثر باسم "مؤشر التأثير" مع شرح مختصر لطريقة احتسابه: هدف، صناعة، تسديد، دفاع، واسترجاع.
- رفع شريط أفضل اللاعبين داخل لوحة المباراة ليظهر قبل الإحصائيات المتقدمة بدل بقائه في الأسفل.
- تحسين لوحة اللاعبين بأيقونات وشرح سريع لكل نوع إحصائية، مع خلفية تقدم بصرية بدل جدول ممل.
- إضافة حركة خفيفة اختيارية للشريط العلوي وخطوط السيطرة بدون رفرفة أو إعادة تحميل مزعجة.
- إضافة وضع جودة بث Ultra يزيد نعومة النص والوضوح البصري داخل OBS.
- إضافة مربع صانع المحتوى في الوسط مع تحكم في الاسم، المعرف، الصورة، والمقياس والموقع.
- إضافة تحكمات سريعة في لوحة الإعدادات لتكبير/تصغير لوحة المباراة ولوحة اللاعبين، وتشغيل/إيقاف الحركة ومربع الصانع.
- حل تداخل إعدادات Match Stats عبر جعل لوحة التحكم السريعة قابلة للتمرير.
- إضافة دقيقة المباراة وحالة النهاية داخل بيانات الاستخراج والقالب.
- إضافة رابط اختياري لكاش صور اللاعبين JSON حتى يمكن تحميل صور اللاعبين من ملف ثابت أو CDN بدل لصق JSON ضخم كل مرة.

## أرشفة المباريات بعد النهاية

أضفت في سيرفر `reo-match-bridge` نظام أرشفة اختياري إلى GitHub. النظام لا يعمل إلا عند وضع هذه المتغيرات في خدمة Google Cloud:

- `REO_ARCHIVE_GITHUB_TOKEN`: توكن GitHub بصلاحية كتابة على المستودع.
- `REO_ARCHIVE_GITHUB_REPO`: اسم المستودع بصيغة `owner/repo`.
- `REO_ARCHIVE_GITHUB_BRANCH`: الفرع، الافتراضي `main`.
- `REO_ARCHIVE_BASE_PATH`: مجلد الأرشيف، الافتراضي `match-archive`.

عند نهاية المباراة يحفظ السيرفر ملف JSON في مسار منظم مثل:

`match-archive/2025-2026/spain-laliga/round-unknown/2026-05-14_1914217_deportivo-alaves-vs-barcelona.json`

الجولة تبقى `round-unknown` إذا لم يوفر WhoScored رقم الجولة داخل البيانات الخام، ويمكن تحسينها لاحقا بإضافة جدول ربط يدوي للمسابقة والجولة.

## كاش صور لاعبي برشلونة من Sortitoutsi

الأفضل هندسيا ألا نربط البث مباشرة بصور خارجية غير مضمونة أو قد تحتاج تسجيل دخول. الطريقة الأكثر ثباتا:

1. اجلب صور اللاعبين المسموح لك باستخدامها واحفظها في مساحة ثابتة مثل Vercel Blob أو Google Cloud Storage أو مستودع GitHub Public.
2. أنشئ ملف JSON بهذه الصيغة:

```json
{
  "Lamine Yamal": "https://your-cdn.example.com/players/barcelona/lamine-yamal.png",
  "Pedri": "https://your-cdn.example.com/players/barcelona/pedri.png",
  "Ferran Torres": "https://your-cdn.example.com/players/barcelona/ferran-torres.png"
}
```

3. ضع رابط هذا الملف في حقل `رابط كاش صور اللاعبين JSON`.
4. يمكن أيضا لصق نفس الخريطة مباشرة في حقل `روابط صور اللاعبين JSON` للحالات السريعة.

القالب يطابق الصور بالاسم أو ID، والإدخال اليدوي يتفوق على رابط الكاش عند وجود الاثنين.

## ملاحظة تشغيلية

ميزة أرشفة GitHub لا تؤثر على OpenClaw ولا تغير خدمته. هي جزء من خدمة `reo-match-bridge` فقط، وتبقى خاملة إذا لم تضف متغيرات GitHub.

## التحقق

- تم تشغيل `npm run lint` بنجاح.
- تم تشغيل `npm run build` بنجاح.
- تم تشغيل `python -m py_compile` على سيرفر الجسر ونسختي الاستخراج بنجاح.
- تم فحص القالب محليا على `http://127.0.0.1:5173` بلقطة متصفح Headless، وظهرت لوحة المباراة واللاعبين ومربع صانع المحتوى بدون قص واضح.
