# تقرير عربي - التخزين والكاش وتثبيت العرض

## أين تخزن بيانات المباريات الآن؟

- في سيرفر Google Cloud تحفظ خدمة `reo-match-bridge` آخر مباراة في:
  `/opt/reo-match-bridge/data/latest-match.json`
- تحفظ رابط المباراة الحالي في:
  `/opt/reo-match-bridge/data/current-match.json`
- في أداة الاستخراج المحلية تحفظ النسخة المباشرة في:
  `extractor/output/latest_match_stats.json`
- وعند تشغيل استخراج يدوي تحفظ ملفات مؤرخة داخل:
  `extractor/output/match_<team>_<timestamp>.json`
- موقع Vercel لا يخزن المباراة بنفسه، بل يقرأ من الجسر عبر:
  `/api/reo-match/match`

## هل يتم التخزين في GitHub الآن؟

ليس تلقائيا إلا إذا تم تفعيل متغيرات الأرشفة في سيرفر Google Cloud. الكود موجود داخل `reo-match-bridge`، لكنه يحتاج:

- `REO_ARCHIVE_GITHUB_TOKEN`
- `REO_ARCHIVE_GITHUB_REPO`
- `REO_ARCHIVE_GITHUB_BRANCH`
- `REO_ARCHIVE_BASE_PATH`

بعد التفعيل، عند نهاية المباراة يحفظ السيرفر الملف في GitHub بمسار منظم مثل:

`match-archive/2025-2026/spain-laliga/round-unknown/2026-05-14_1914217_deportivo-alaves-vs-barcelona.json`

تم تحسين الأرشفة الآن لتحدث الملف إذا كان موجودا سابقا بدل أن تفشل بسبب تكرار المسار.

## سبب التقطيع في العرض

السبب الأقرب كان في نافذة العرض نفسها: fallback polling كان سريع جدا، وإذا لم تصل نسخة واضحة من السيرفر كان يعيد جلب الحالة كاملة ويطبق نفس القالب مرارا. هذا يعطي إحساس رفرش حتى لو البيانات لم تتغير.

تم تخفيف ذلك عبر:

- منع تطبيق نفس حالة القالب مرتين إذا لم تتغير فعليا.
- جعل fallback polling أهدأ: 2.5 ثانية في OBS و3 ثواني في المتصفح.
- إلغاء polling المكرر عبر `requestAnimationFrame`.
- تخفيف حركة اللمعان من 7 ثوان إلى 18 ثانية، وحركة خطوط السيطرة من 2.6 ثانية إلى 9 ثوان بدون تمدد واضح.

## نظام كاش صور اللاعبين

أضفت ملف بداية هنا:

`public/player-image-cache/barcelona.json`

والقالب الآن يقرأه افتراضيا من:

`/player-image-cache/barcelona.json`

الملف حاليا يحتوي أسماء لاعبي برشلونة بدون روابط. عند وضع روابط صور مباشرة صالحة، سيعرضها القالب تلقائيا. الصيغة:

```json
{
  "Lamine Yamal": "https://your-cdn.example.com/players/barcelona/lamine-yamal.png",
  "Pedri": "https://your-cdn.example.com/players/barcelona/pedri.png"
}
```

الأفضل ألا يتم hotlink مباشرة من Sortitoutsi أثناء البث. الطريقة الآمنة:

1. تحميل الصور المسموح باستخدامها.
2. رفعها إلى مساحة ثابتة تملكها مثل GitHub public raw أو Google Cloud Storage أو Vercel Blob.
3. وضع الروابط النهائية داخل `public/player-image-cache/barcelona.json`.
4. القالب يطابق بالاسم أو ID، والإدخال اليدوي في `playerImageMapJson` يتفوق على الكاش إذا أردت تغيير صورة مؤقتا.
